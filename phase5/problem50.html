#50 — Longest Palindromic Substring 🏆

🔖 Pattern: DP / Expand Around Center

🎯 Real-World Scenario THE FINALE ▼

Tum ek text editor bana rahe ho with a "find palindrome" feature:

Input: "babad"
Palindromes: "b","a","bab","aba","d"
Longest: "bab" or "aba" (length 3)

Input: "cbbd"
Longest: "bb" (length 2)

This is LeetCode #5 — Longest Palindromic Substring. The FINAL problem of our 50-problem roadmap.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Find the longest substring (contiguous) that is a palindrome
  • Palindrome = reads same forwards and backwards
  • Return the actual substring, not just length

Three approaches:

Approach Time Space Idea
Brute ForceO(n³)O(1)Check every substring
Expand Around CenterO(n²)O(1)Expand from each center
DPO(n²)O(n²)dp[i][j] = is s[i..j] palindrome?

Expand Around Center is the best — O(n²) time with O(1) space. Simpler than DP too.

💪 2. Brute Force Thinking STEP 2 ▼

Check every substring: O(n²) substrings × O(n) palindrome check = O(n³).

🤔 Key insight: every palindrome has a CENTER. Expand outward from each possible center. If chars match → keep expanding. If not → stop.

⚡ 3. Optimization — Expand Around Center STEP 3 ▼

Two types of palindrome centers:

  • Odd length: center is one character. "aba" → center at 'b'
  • Even length: center is between two characters. "abba" → center between 'b','b'

Algorithm:

  1. For each index i (0 to n-1):
    • Expand from center (i, i) → odd-length palindromes
    • Expand from center (i, i+1) → even-length palindromes
  2. Expand: while left ≥ 0 AND right < n AND s[left] == s[right] → expand (left--, right++)
  3. Track the longest palindrome found
int expand(String s, int left, int right) {
    while (left >= 0 && right < s.length()
           && s.charAt(left) == s.charAt(right)) {
        left--;
        right++;
    }
    return right - left - 1; // length of palindrome
}

💡 Pattern: Expand Around Center — for each of 2n-1 centers, expand outward while chars match. O(n²) time, O(1) space.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why 2n-1 centers?

  • n characters = n odd centers (each character)
  • n-1 gaps between characters = n-1 even centers
  • Total: n + (n-1) = 2n-1 centers
  • Each center expands at most O(n) → total O(n × n) = O(n²)

DP approach (alternative):

dp[i][j] = true if s[i..j] is palindrome
dp[i][i] = true (single char)
dp[i][i+1] = (s[i] == s[i+1]) (two chars)
dp[i][j] = dp[i+1][j-1] && s[i]==s[j] (expand from inner)

DP uses O(n²) space. Expand Around Center uses O(1). Both O(n²) time. Expand is preferred.

☕ 8. Java Implementation STEP 8 ▼ 
public class LongestPalindrome {

    public static String longestPalindrome(String s) {
        int start = 0, maxLen = 0;

        for (int i = 0; i < s.length(); i++) {
            int odd  = expand(s, i, i);     // odd-length
            int even = expand(s, i, i + 1); // even-length
            int len = Math.max(odd, even);

            if (len > maxLen) {
                maxLen = len;
                start = i - (len - 1) / 2;
            }
        }
        return s.substring(start, start + maxLen);
    }

    private static int expand(String s, int left, int right) {
        while (left >= 0 && right < s.length()
               && s.charAt(left) == s.charAt(right)) {
            left--;
            right++;
        }
        return right - left - 1;
    }

    public static void main(String[] args) {
        System.out.println(longestPalindrome("babad")); // "bab"
        System.out.println(longestPalindrome("cbbd"));  // "bb"
    }
}

💡 start = i - (len-1)/2 — calculates the start index of the palindrome from its center and length. Works for both odd and even.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting even-length palindromes

    ❌ Only expanding (i,i) → misses "bb", "abba"

    ✅ Must expand BOTH (i,i) for odd AND (i,i+1) for even

  2. Wrong start index calculation

    ✅ start = i - (len-1)/2 — integer division handles both odd and even correctly

  3. Returning length instead of actual substring

    ✅ Track start and maxLen, return s.substring(start, start+maxLen)
