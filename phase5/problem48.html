#48 — Burst Balloons for Max Points

🔖 Pattern: DP (Interval DP)

🎯 Real-World Scenario START HERE ▼

Tum ek carnival game khel rahe ho. Balloons hain with numbers: [3, 1, 5, 8]

Jab tum balloon i burst karte ho, points milte hain: nums[i-1] × nums[i] × nums[i+1] (left neighbor × self × right neighbor). Boundary pe 1 assume karo.

"Maximize total points by bursting all balloons in optimal order."

Burst 1: 3×1×5 = 15  → [3, 5, 8]
Burst 5: 3×5×8 = 120 → [3, 8]
Burst 3: 1×3×8 = 24  → [8]
Burst 8: 1×8×1 = 8   → []
Total: 15+120+24+8 = 167

But optimal order gives 167. Is there better? Yes!
Optimal: burst order [1,5,3,8] → total = 167

This is LeetCode #312 — Burst Balloons. Hard-rated. The classic Interval DP problem.

🧠 1. Problem Understanding STEP 1 ▼

What is Interval DP?

  • Previous DP: dp[i] (1D), dp[i][j] where i=item and j=capacity or i=string1 and j=string2
  • Interval DP: dp[i][j] = best answer for the range [i..j]
  • We try every possible "split point" k within the range and combine left and right subproblems

The key trick — think about the LAST balloon to burst:

  • Instead of "which balloon to burst first?" (hard — neighbors change)
  • Think: "which balloon is burst LAST in range [i..j]?" (easier — boundaries are fixed)
  • If balloon k is burst last in [i..j], its neighbors are i-1 and j+1 (everything else already gone)
  • Points for k = nums[i-1] × nums[k] × nums[j+1]
  • Plus: best points from [i..k-1] and [k+1..j] (already burst optimally)

DP recipe:

  1. State: dp[i][j] = max points from bursting all balloons in range [i..j]
  2. Recurrence: dp[i][j] = max over k in [i..j] of: dp[i][k-1] + nums[i-1]×nums[k]×nums[j+1] + dp[k+1][j]
  3. Base: dp[i][j] = 0 when i > j (empty range)
  4. Order: Fill by increasing range length (len=1, then 2, then 3...)
  5. Answer: dp[1][n] (entire range)
⚡ 3. Optimization Thinking STEP 3 ▼

The "last to burst" insight:

Range [i..j], balloon k is burst LAST:

Left subproblem: [i..k-1] — burst optimally (k is still there as right boundary)
Right subproblem: [k+1..j] — burst optimally (k is still there as left boundary)
Finally burst k: neighbors are i-1 and j+1 (everything else in [i..j] is gone)

dp[i][j] = max for k = i to j:
    dp[i][k-1] + nums[i-1] * nums[k] * nums[j+1] + dp[k+1][j]

Pad the array with 1s: Add 1 at both ends: [1, 3, 1, 5, 8, 1]. This handles boundary conditions cleanly — no special cases for first/last balloon.

💡 Pattern: Interval DP — dp[i][j] for range, try every split point k, combine left + right + cost of k. Fill by increasing range length.

🔍 5. Step-by-Step Dry Run STEP 5 ▼

nums = [3,1,5,8] → padded: [1, 3, 1, 5, 8, 1] (indices 0-5)

Range length 1 (single balloons):

  • dp[1][1]: k=1 → 1×3×1 = 3
  • dp[2][2]: k=2 → 3×1×5 = 15
  • dp[3][3]: k=3 → 1×5×8 = 40
  • dp[4][4]: k=4 → 5×8×1 = 40

Range length 2:

  • dp[1][2]: k=1 → 1×3×5 + dp[2][2] = 15+15=30. k=2 → dp[1][1] + 1×1×5 = 3+5=8. max=30
  • dp[3][4]: k=3 → 1×5×1 + dp[4][4] = 5+40=45. k=4 → dp[3][3] + 1×8×1 = 40+8=48. max=48

... continuing to range length 4:

dp[1][4] = 167 ✓

The key: we fill small ranges first, then use them to compute larger ranges. Each range tries every possible "last balloon to burst."

⚙️ 6. Internal Mechanics STEP 6 ▼

Interval DP template:

// Fill by increasing range length
for len = 1 to n:
    for i = 1 to n - len + 1:
        j = i + len - 1
        for k = i to j:  // try every split point
            dp[i][j] = max(dp[i][j],
                dp[i][k-1] + cost(i,k,j) + dp[k+1][j])

Interval DP problems:

  • Burst Balloons (this) — last to burst in range
  • Matrix Chain Multiplication — optimal parenthesization
  • Minimum Cost to Merge Stones
  • Palindrome Partitioning II

All share the same structure: dp[i][j] for range, try split points, combine subranges.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(n³)
Space:O(n²)

Three nested loops: range length × start position × split point. dp table is n×n.

☕ 8. Java Implementation STEP 8 ▼ 
public class BurstBalloons {

    public static int maxCoins(int[] nums) {
        int n = nums.length;
        int[] arr = new int[n + 2]; // pad with 1s
        arr[0] = arr[n + 1] = 1;
        for (int i = 0; i < n; i++) arr[i + 1] = nums[i];

        int[][] dp = new int[n + 2][n + 2];

        for (int len = 1; len <= n; len++) {
            for (int i = 1; i <= n - len + 1; i++) {
                int j = i + len - 1;
                for (int k = i; k <= j; k++) {
                    int coins = dp[i][k - 1]
                              + arr[i - 1] * arr[k] * arr[j + 1]
                              + dp[k + 1][j];
                    dp[i][j] = Math.max(dp[i][j], coins);
                }
            }
        }
        return dp[1][n];
    }

    public static void main(String[] args) {
        System.out.println(maxCoins(new int[]{3,1,5,8})); // 167
    }
}
⚠️ 10. Common Mistakes STEP 10 ▼
  1. Thinking "first to burst" instead of "last to burst"

    ❌ First-to-burst changes neighbors → subproblems aren't independent

    ✅ Last-to-burst keeps boundaries fixed → clean subproblem decomposition

  2. Forgetting to pad with 1s

    ✅ arr = [1, ...nums..., 1] — handles boundary balloons cleanly

  3. Wrong fill order

    ✅ Must fill small ranges before large (len=1, then 2, then 3...)

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when: ✅ "Optimal order to process a range" ✅ "Matrix chain multiplication" ✅ "Merge stones" ✅ Any problem where you split a range and combine subranges

Don't use when: ❌ No range structure (use other DP types) ❌ n > 500 (O(n³) too slow)
