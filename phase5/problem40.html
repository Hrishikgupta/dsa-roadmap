#40 — Minimum Cost to Reach Home

🔖 Pattern: DP (Min Cost Path — 2D Grid)

🎯 Real-World Scenario START HERE ▼

Tum ek city grid mein ho. Har intersection pe toll hai. Top-left se bottom-right (home) tak jaana hai with minimum toll cost. Only move right or down.

Toll grid:
  1  3  1
  1  5  1
  4  2  1

Cheapest path: 1→3→1→1→1 = 7
1 → 3 → 1
              ↓
              1
              ↓
              1  = total cost 7

This is LeetCode #64 — Minimum Path Sum. Classic 2D DP introduction.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an m×n grid with non-negative costs, find the path from top-left to bottom-right with minimum total cost
  • Can only move right or down

Applying the DP recipe from #39:

  1. State: dp[i][j] = minimum cost to reach cell (i,j)
  2. Recurrence: dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1]) — came from above or left
  3. Base cases: First row: can only come from left. First column: can only come from above.
  4. Order: Fill row by row, left to right (each cell depends on top and left)
  5. Answer: dp[m-1][n-1]

Connection to #39:

  • #39: 1D DP — dp[i] = f(dp[i-1], dp[i-2])
  • #40: 2D DP — dp[i][j] = f(dp[i-1][j], dp[i][j-1]) — same idea, two dimensions
💪 2. Brute Force Thinking STEP 2 ▼

Recursion: At each cell, try going right and going down. Take the minimum.

int minPath(grid, i, j) {
    if (i == 0 && j == 0) return grid[0][0];
    if (i < 0 || j < 0) return Integer.MAX_VALUE;
    return grid[i][j] + Math.min(minPath(grid,i-1,j), minPath(grid,i,j-1));
}

O(2^(m+n)) — exponential. Overlapping subproblems: cell (1,1) is reached from (0,1) and (1,0), computed twice.

🤔 Same fix as #39: memoize or build bottom-up.

⚡ 3. Optimization Thinking STEP 3 ▼

Bottom-up DP:

// Fill first row (can only come from left)
for j = 1 to n-1: dp[0][j] = dp[0][j-1] + grid[0][j]

// Fill first column (can only come from above)
for i = 1 to m-1: dp[i][0] = dp[i-1][0] + grid[i][0]

// Fill rest (min of top and left)
for i = 1 to m-1:
  for j = 1 to n-1:
    dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])

Space optimization: Since each row only depends on the current and previous row, we can use a 1D array and update in-place (left to right). But the 2D version is clearer for interviews.

💡 Pattern: 2D Grid DP — each cell = cost + min(top, left). Fill row by row.

🎯 4. Final Approach STEP 4 ▼

Can modify grid in-place (no extra dp array needed):

  • grid[i][j] += min(grid[i-1][j], grid[i][j-1])
  • After processing, grid[m-1][n-1] = answer
  • O(m×n) time, O(1) extra space

Grid: [[1,3,1],[1,5,1],[4,2,1]]

⚙️ 6. Internal Mechanics STEP 6 ▼

1D DP (#39) vs 2D DP (#40):

Aspect #39 Climbing Stairs #40 Min Cost Path
Statedp[i] (1 variable)dp[i][j] (2 variables)
Depends ondp[i-1], dp[i-2]dp[i-1][j], dp[i][j-1]
Basedp[1]=1, dp[2]=2First row, first column
Fill orderLeft to rightRow by row, left to right

2D Grid DP family:

  • Min path sum (this) — min cost, move right/down
  • Unique paths — count paths, move right/down
  • Unique paths with obstacles — same but skip blocked cells
  • Maximal square — largest square of 1s in binary matrix
📊 7. Complexity Analysis STEP 7 ▼
Time:O(m × n)
Space:O(1)*

*O(1) if modifying grid in-place. O(m×n) if using separate dp array. O(n) with rolling array optimization.

☕ 8. Java Implementation STEP 8 ▼ 
public class MinPathSum {

    public static int minPathSum(int[][] grid) {
        int m = grid.length, n = grid[0].length;

        // Fill first row
        for (int j = 1; j < n; j++)
            grid[0][j] += grid[0][j - 1];

        // Fill first column
        for (int i = 1; i < m; i++)
            grid[i][0] += grid[i - 1][0];

        // Fill rest
        for (int i = 1; i < m; i++)
            for (int j = 1; j < n; j++)
                grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);

        return grid[m - 1][n - 1];
    }
}

💡 In-place modification: no extra dp array. grid itself becomes the dp table. Ask interviewer if modifying input is OK.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting to handle first row and first column separately

    ❌ dp[0][j] = min(dp[-1][j], dp[0][j-1]) → index out of bounds

    ✅ First row: only from left. First column: only from above. No min needed.

  2. Using max instead of min (or vice versa)

    Read carefully: "minimum cost" → use min. "Maximum path sum" → use max.

  3. Allowing up/left movement

    This problem only allows right/down. If all 4 directions allowed → need BFS/Dijkstra, not DP.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Min/max cost path in grid" with restricted movement (right/down only)
  • ✅ "Count unique paths" in a grid
  • ✅ Any grid problem where movement is monotonic (only right/down, never back)
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Movement in all 4 directions — use BFS (#32) or Dijkstra (#35)
  • ❌ Grid has negative costs with cycles — DP doesn't work (need Bellman-Ford)
  • ❌ "Find the actual path" (not just cost) — need to trace back through dp table
💡 "2D Grid DP: dp[i][j] = cost + min(top, left). First row/column are base cases (only one direction). Same DP recipe as 1D, just two indices."
