#42 — Longest Increasing Stock Prices

🔖 Pattern: DP (LIS — Longest Increasing Subsequence)

🎯 Real-World Scenario START HERE ▼

Tum ek stock analyst ho. Daily closing prices hain:

[10, 9, 2, 5, 3, 7, 101, 18]

Boss bola: "Sabse lamba increasing trend dhundho — consecutive nahi, subsequence."

  • [2, 5, 7, 101] → length 4 ✓ (increasing subsequence)
  • [2, 3, 7, 18] → length 4 ✓
  • [2, 5, 7, 18] → length 4 ✓

Answer: 4 (longest increasing subsequence length)

This is LeetCode #300 — Longest Increasing Subsequence. Classic DP with O(n log n) optimization.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Find the length of the longest strictly increasing subsequence
  • Subsequence = elements in original order, but not necessarily contiguous
  • [2,5,7,101] from [10,9,2,5,3,7,101,18] — skip elements, keep order

How this differs from previous DP:

Problem dp[i] depends on Transition
#39 Stairsdp[i-1], dp[i-2]Fixed neighbors
#41 Robberdp[i-1], dp[i-2]Fixed neighbors
#42 LISALL dp[j] where j < iScan all previous elements

💡 LIS is the first DP where each state depends on ALL previous states (not just 1-2 neighbors). This makes it O(n²).

💪 2. Brute Force Thinking STEP 2 ▼

Generate all subsequences, check if increasing, track max length.

2ⁿ subsequences → O(2ⁿ). Way too slow.

🤔 DP approach: for each element, find the longest increasing subsequence ENDING at that element.

⚡ 3. Optimization Thinking STEP 3 ▼

O(n²) DP approach:

  • dp[i] = length of LIS ending at index i
  • For each i, look at ALL j < i where nums[j] < nums[i]
  • dp[i] = max(dp[j] + 1) for all valid j
  • Base: dp[i] = 1 (element itself is a subsequence of length 1)
for i = 0 to n-1:
    dp[i] = 1
    for j = 0 to i-1:
        if nums[j] < nums[i]:
            dp[i] = max(dp[i], dp[j] + 1)
answer = max(dp[0]...dp[n-1])

O(n log n) optimization (Binary Search + Patience Sorting):

  • Maintain a "tails" array: tails[i] = smallest tail element of all increasing subsequences of length i+1
  • For each element: binary search in tails to find where it fits
  • If larger than all tails → append (extend longest subsequence)
  • Else → replace the first tail that's ≥ current element (keep tails as small as possible)
  • Length of tails array = LIS length

💡 Two approaches: O(n²) DP (intuitive, good for interviews) and O(n log n) binary search (optimal, impressive to show).

🎯 4. Final Approach STEP 4 ▼

DP recipe:

  1. State: dp[i] = LIS length ending at index i
  2. Recurrence: dp[i] = 1 + max(dp[j]) for all j < i where nums[j] < nums[i]
  3. Base: dp[i] = 1 for all i
  4. Order: Left to right (each i scans all j < i)
  5. Answer: max(dp[0..n-1]) — NOT dp[n-1]! LIS might not end at the last element.

⚙️ 6. Internal Mechanics STEP 6 ▼

O(n log n) Binary Search approach — how it works:

Process [10, 9, 2, 5, 3, 7, 101, 18]:
tails = []

10: tails=[] → append → [10]
 9: 9 < 10 → replace 10 → [9]
 2: 2 < 9  → replace 9  → [2]
 5: 5 > 2  → append    → [2, 5]
 3: 3 < 5  → replace 5 → [2, 3]
 7: 7 > 3  → append    → [2, 3, 7]
101: 101>7 → append    → [2, 3, 7, 101]
18: 18<101 → replace   → [2, 3, 7, 18]

tails.length = 4 = LIS length ✓

Note: tails array is NOT the actual LIS — it's a helper structure. Its LENGTH equals the LIS length. Binary search finds the replacement position in O(log n).

O(n²) vs O(n log n):

☕ 8. Java Implementation STEP 8 ▼

O(n²) DP:

public static int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1);
    int maxLen = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i])
                dp[i] = Math.max(dp[i], dp[j] + 1);
        }
        maxLen = Math.max(maxLen, dp[i]);
    }
    return maxLen;
}

O(n log n) Binary Search:

public static int lengthOfLIS_BS(int[] nums) {
    List<Integer> tails = new ArrayList<>();
    for (int num : nums) {
        int pos = Collections.binarySearch(tails, num);
        if (pos < 0) pos = -(pos + 1);
        if (pos == tails.size()) tails.add(num);
        else tails.set(pos, num);
    }
    return tails.size();
}

💡 Collections.binarySearch returns negative insertion point if not found. Convert with -(pos+1).

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Returning dp[n-1] instead of max(dp)

    ❌ LIS might not end at the last element. [1,3,2] → LIS=[1,3] ends at index 1, not 2.

    ✅ Answer = max of entire dp array

  2. Using ≤ instead of < for strictly increasing

    ❌ [1,1,1] with ≤ gives LIS=3 (wrong — equal is not increasing)

    ✅ Use strict < for strictly increasing

  3. Confusing subsequence with subarray

    Subsequence: elements in order, not necessarily contiguous. Subarray: contiguous.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Longest increasing/decreasing subsequence"
  • ✅ "Minimum number of deletions to make sorted"
  • ✅ "Russian doll envelopes" (2D LIS)
  • ✅ "Longest chain of pairs"
  • ✅ Any problem asking for longest subsequence with a monotonic property
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Longest increasing SUBARRAY" (contiguous) — use simple scan, not DP
  • ❌ "Find the actual LIS" (not just length) — need to trace back through dp or maintain parent pointers
💡 "LIS: dp[i] = 1 + max(dp[j]) for all j < i where nums[j] < nums[i]. Answer = max of entire dp array. O(n²) DP or O(n log n) with binary search on tails."
