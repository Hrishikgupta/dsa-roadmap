#43 — Knapsack — Packing for a Trip

🔖 Pattern: DP (0/1 Knapsack)

🎯 Real-World Scenario START HERE ▼

Tum trip pe ja rahe ho. Bag capacity = 7 kg. Items available:

ItemWeightValue
Laptop3 kg₹4
Camera4 kg₹5
Book2 kg₹3

Question: "Maximum value kya pack kar sakte ho within 7 kg?"

Laptop(3kg,₹4) + Camera(4kg,₹5) = 7kg, ₹9. Or Laptop + Book = 5kg, ₹7. Best: ₹9

The 0/1 Knapsack — the most important DP pattern. Foundation for Coin Change, Subset Sum, Partition, and many more.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given n items, each with weight and value, and a bag with capacity W
  • Each item can be taken at most once (0/1 = take or don't take)
  • Maximize total value without exceeding capacity

Why "0/1"? Each item: 0 (don't take) or 1 (take). No fractions, no duplicates.

DP recipe:

  1. State: dp[i][w] = max value using first i items with capacity w
  2. Recurrence:
    • If wt[i] > w: can't take item i → dp[i][w] = dp[i-1][w]
    • Else: dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i])
  3. Base: dp[0][w] = 0 (no items), dp[i][0] = 0 (no capacity)
  4. Order: Row by row (item by item), left to right (capacity 0 to W)
  5. Answer: dp[n][W]

Connection to #41 (House Robber):

  • #41: Include/Exclude with adjacency constraint → 1D DP
  • #43: Include/Exclude with capacity constraint → 2D DP (items × capacity)
  • Same include/exclude decision, but now "capacity remaining" is a second dimension
💪 2. Brute Force Thinking STEP 2 ▼

Try all 2ⁿ subsets (#21), check weight ≤ W, track max value. O(2ⁿ).

🤔 DP reduces this to O(n × W) by remembering: "what's the best value I can get with the first i items and capacity w?"

⚡ 3. Optimization Thinking STEP 3 ▼

The include/exclude decision at each item:

For item i with weight wt[i] and value val[i]:

EXCLUDE: don't take item i
  dp[i][w] = dp[i-1][w]  // same value as without this item

INCLUDE: take item i (only if wt[i] ≤ w)
  dp[i][w] = dp[i-1][w - wt[i]] + val[i]  // value of item + best with remaining capacity

CHOOSE: max of both
  dp[i][w] = max(exclude, include)

Space optimization: Since row i only depends on row i-1, we can use a 1D array and fill RIGHT TO LEFT (to avoid using updated values from the same row).

// 1D space-optimized
for each item i:
    for w = W down to wt[i]:  // RIGHT TO LEFT!
        dp[w] = max(dp[w], dp[w - wt[i]] + val[i])

💡 Pattern: 0/1 Knapsack — 2D DP (items × capacity). Include/Exclude at each item. Space-optimize to 1D by iterating capacity RIGHT TO LEFT.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why RIGHT TO LEFT in 1D optimization:

LEFT TO RIGHT (WRONG for 0/1):
dp[3] = max(dp[3], dp[3-3]+4) = max(0, 0+4) = 4  // take laptop
dp[6] = max(dp[6], dp[6-3]+4) = max(0, 4+4) = 8  // takes laptop AGAIN! (dp[3] already updated)

RIGHT TO LEFT (CORRECT):
dp[7] = max(dp[7], dp[7-3]+4) = max(0, dp[4]+4)  // dp[4] still from previous row
dp[3] = max(dp[3], dp[0]+4) = 4  // dp[0] still from previous row

Right-to-left ensures we read from "previous row" values (not yet updated in this round). This prevents using an item twice.

Knapsack family:

Variant Items 1D loop direction Problem
0/1 KnapsackEach used at most onceRIGHT TO LEFT#43 (this), #45
Unbounded KnapsackUnlimited copiesLEFT TO RIGHT#44 (Coin Change)

The ONLY difference between 0/1 and unbounded in 1D: loop direction. Right-to-left = use once. Left-to-right = use unlimited.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(n × W)
Space:O(W)

n items × W capacity. Space O(W) with 1D optimization, O(n×W) with 2D table.

☕ 8. Java Implementation STEP 8 ▼

1D space-optimized (RIGHT TO LEFT):

public class Knapsack01 {

    public static int knapsack(int[] wt, int[] val, int W) {
        int[] dp = new int[W + 1];

        for (int i = 0; i < wt.length; i++) {
            for (int w = W; w >= wt[i]; w--) { // RIGHT TO LEFT!
                dp[w] = Math.max(dp[w], dp[w - wt[i]] + val[i]);
            }
        }
        return dp[W];
    }

    public static void main(String[] args) {
        int[] wt = {3, 4, 2}, val = {4, 5, 3};
        System.out.println(knapsack(wt, val, 7)); // 9
    }
}

💡 The inner loop goes from W DOWN TO wt[i]. This single direction change is what makes it 0/1 (not unbounded). Memorize this.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Iterating LEFT TO RIGHT in 1D 0/1 knapsack

    ❌ Allows using same item multiple times → becomes unbounded knapsack

    ✅ RIGHT TO LEFT for 0/1. LEFT TO RIGHT for unbounded (#44).

  2. Confusing 0/1 with unbounded

    0/1: each item once. Unbounded: unlimited copies. The ONLY code difference in 1D = loop direction.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Maximize value with weight/capacity limit" — classic knapsack
  • ✅ "Subset sum" (#45) — knapsack where value = weight, target = specific sum
  • ✅ "Partition equal subset sum" (#45) — can we split into two equal halves?
  • ✅ "Target sum" — assign +/- to elements to reach target
  • ✅ Any "select items with constraint, optimize value" problem
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Unlimited copies of items → use Unbounded Knapsack (#44)
  • ❌ W is very large (>10⁶) → O(n×W) is too slow, need different approach
  • ❌ Fractional items allowed → use Greedy (take highest value/weight ratio first)
💡 "0/1 Knapsack: for each item, max(skip, take + best with remaining capacity). 1D optimization: iterate capacity RIGHT TO LEFT to prevent reuse. This pattern is the foundation of half of all DP problems."
