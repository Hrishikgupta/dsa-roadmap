#41 — Robbing Houses Without Getting Caught

🔖 Pattern: DP (Include/Exclude)

🎯 Real-World Scenario START HERE ▼

Tum ek (fictional!) thief ho. Street pe houses hain with cash:

[2, 7, 9, 3, 1]

Rule: Adjacent houses have connected alarms. If you rob two adjacent houses, alarm triggers.

Question: "Maximum kitna paisa loot sakte ho without triggering alarm?"

  • Rob houses 0,2,4: 2+9+1 = 12
  • Rob houses 1,3: 7+3 = 10
  • Rob houses 0,2: 2+9 = 11
  • Rob houses 1,2? NO — adjacent!

Answer: 12 (houses 0, 2, 4)

This is LeetCode #198 — House Robber. Classic Include/Exclude DP.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given array of non-negative integers (money in each house)
  • Can't rob two adjacent houses
  • Maximize total money robbed

The Include/Exclude decision at each house:

  • Rob house i (include): take nums[i] + best from house i-2 (skip neighbor)
  • Skip house i (exclude): take best from house i-1 (as if house i doesn't exist)
  • Take the max of both choices

Recurrence: dp[i] = max(dp[i-1], dp[i-2] + nums[i])

Connection to previous:

  • #21 (Subsets): Include/Exclude recursion → generate all subsets
  • #41 (House Robber): Include/Exclude DP → OPTIMIZE (find max, not generate all)
  • Same decision tree, but instead of exploring all branches, we pick the best at each step
💪 2. Brute Force Thinking STEP 2 ▼

Recursion — try all valid subsets:

int rob(int[] nums, int i) {
    if (i < 0) return 0;
    return Math.max(
        rob(nums, i - 1),              // skip house i
        rob(nums, i - 2) + nums[i]    // rob house i
    );
}

O(2ⁿ) — same overlapping subproblems as Climbing Stairs. rob(3) gets called from rob(5) and rob(4).

🤔 Same fix: memoize or bottom-up. And since dp[i] only depends on dp[i-1] and dp[i-2] → space optimize to O(1).

⚡ 3. Optimization Thinking STEP 3 ▼

Bottom-up DP:

dp[0] = nums[0]
dp[1] = max(nums[0], nums[1])
for i = 2 to n-1:
    dp[i] = max(dp[i-1], dp[i-2] + nums[i])

Space-optimized (two variables):

int prev2 = 0, prev1 = 0;
for (int num : nums) {
    int curr = Math.max(prev1, prev2 + num);
    prev2 = prev1;
    prev1 = curr;
}
return prev1;

O(n) time, O(1) space. Same pattern as Climbing Stairs (#39) — Fibonacci-like with a twist (max instead of sum).

💡 Pattern: Include/Exclude DP — at each element: max(skip it, take it + best from 2 back). Fibonacci structure with optimization.

🎯 4. Final Approach STEP 4 ▼

DP recipe:

  1. State: dp[i] = max money robbing from houses 0 to i
  2. Recurrence: dp[i] = max(dp[i-1], dp[i-2] + nums[i])
  3. Base: dp[0] = nums[0], dp[1] = max(nums[0], nums[1])
  4. Order: Left to right
  5. Answer: dp[n-1]

💡 The space-optimized version (prev2, prev1) is the cleanest. prev2 = dp[i-2], prev1 = dp[i-1].

⚙️ 6. Internal Mechanics STEP 6 ▼

Include/Exclude DP pattern — appears in many problems:

Problem Include Exclude Combine
House Robber (this)dp[i-2] + nums[i]dp[i-1]max
0/1 Knapsack (#43)dp[i-1][w-wt] + valdp[i-1][w]max
Subset Sum (#45)dp[i-1][s-nums[i]]dp[i-1][s]OR

Same structure: for each item, decide include or exclude. The combine function (max, min, OR, +) changes per problem.

House Robber variants:

  • House Robber I (this) — linear street
  • House Robber II (LC #213) — circular street (first and last are adjacent)
  • House Robber III (LC #337) — houses form a tree
📊 7. Complexity Analysis STEP 7 ▼
Time:O(n)
Space:O(1)

Single pass. Two variables (prev1, prev2). Same as Climbing Stairs optimization.

☕ 8. Java Implementation STEP 8 ▼ 
public class HouseRobber {

    public static int rob(int[] nums) {
        int prev2 = 0, prev1 = 0;

        for (int num : nums) {
            int curr = Math.max(prev1, prev2 + num);
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }

    public static void main(String[] args) {
        System.out.println(rob(new int[]{2,7,9,3,1})); // 12
        System.out.println(rob(new int[]{1,2,3,1})); // 4
    }
}

💡 4 lines of logic. prev2 = "best if I robbed 2 houses ago", prev1 = "best up to previous house". curr = max(skip, rob).

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Thinking you must rob every other house

    ❌ You CAN skip multiple houses in a row. [1,100,1,1,100] → rob houses 1 and 4 (skip 2,3)

    ✅ The DP naturally handles this — "skip" means take dp[i-1] which might have skipped too

  2. Greedy approach (always rob the biggest)

    ❌ [2,1,1,2] → greedy picks 2,2=4 but optimal is 2+1=3? No, 2+2=4 is correct here. But [1,3,1,3,1] → greedy picks 3,3=6 but 1+1+1=3 is worse. Greedy works here but fails on [2,7,9,3,1].

    ✅ DP is the only correct approach — greedy doesn't generalize

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Max/min value with no-adjacent constraint"
  • ✅ "Include or exclude each element" with optimization
  • ✅ "Delete and earn" (LC #740) — same pattern, group by value
  • ✅ Any problem where taking an item "blocks" nearby items
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ No adjacency constraint — just sum everything (no DP needed)
  • ❌ Circular arrangement (House Robber II) — need to run DP twice (exclude first or exclude last)
  • ❌ Tree structure (House Robber III) — need tree DP, not linear DP
💡 "Include/Exclude DP: dp[i] = max(skip it = dp[i-1], take it = dp[i-2] + nums[i]). Same Fibonacci structure as Climbing Stairs, but with max instead of sum."
