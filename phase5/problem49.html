#49 — Word Break — Can This Sentence Be Split?

🔖 Pattern: DP + Hashing

🎯 Real-World Scenario START HERE ▼

Tum ek search engine bana rahe ho. User ne bina spaces ke type kiya: "leetcode"

Dictionary: ["leet", "code"]

"Kya yeh string dictionary words mein split ho sakti hai?"

  • "leetcode" → "leet" + "code" → YES ✓
  • "applepenapple" with dict ["apple","pen"] → "apple" + "pen" + "apple" → YES ✓
  • "catsandog" with dict ["cats","dog","sand","and","cat"] → NO ✗

This is LeetCode #139 — Word Break. Combines DP (#39 pattern) with HashSet (#2 pattern).

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a string and a dictionary of words, can the string be segmented into dictionary words?
  • Words can be reused (unlimited — like unbounded knapsack!)

DP + HashSet combo:

  • HashSet (from #2): O(1) lookup for "is this substring a dictionary word?"
  • DP (from #39): dp[i] = "can s[0..i-1] be segmented into dictionary words?"

DP recipe:

  1. State: dp[i] = can s[0..i-1] be segmented? (boolean)
  2. Recurrence: dp[i] = true if there exists j < i where dp[j] == true AND s[j..i-1] is in dictionary
  3. Base: dp[0] = true (empty string is "segmented")
  4. Order: Left to right (i = 1 to n)
  5. Answer: dp[n]

Intuition: For each position i, check all possible last words ending at i. If any of them starts at a position j where dp[j] is true → dp[i] is true.

⚡ 3. Optimization Thinking STEP 3 ▼

Optimization: limit j range by max word length:

  • Instead of j from 0 to i-1, only check j from i - maxWordLen to i-1
  • No dictionary word is longer than maxWordLen → earlier j's can't form a valid word

💡 Pattern: DP + HashSet — dp[i] checks all possible last words ending at i. HashSet gives O(1) word lookup. Break early on first valid split.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why "catsandog" fails with dict ["cats","dog","sand","and","cat"]:

  • dp[3] = true ("cat"), dp[4] = true ("cats")
  • dp[7] = true ("cats" + "and" or "cat" + "sand")
  • But dp[8]? Need word ending at 8. s[7..8] = "o" ∉ dict. s[4..8] = "ando" ∉ dict. No valid split.
  • dp[10]? s[7..10] = "dog" ∈ dict, but dp[7] = true → dp[10] = true? Wait, s = "catsandog" has length 9.
  • dp[9]: s[7..9] = "og" ∉ dict. s[4..9] = "andog" ∉ dict. No valid split → false.

Word Break variants:

  • Word Break I (this) — can it be segmented? (boolean)
  • Word Break II (LC #140) — return ALL valid segmentations (backtracking + DP)
📊 7. Complexity Analysis STEP 7 ▼
Time:O(n² × k)
Space:O(n)

n = string length, k = average substring comparison cost. With max word length optimization: O(n × maxLen × k). HashSet lookup is O(k) for string of length k.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class WordBreak {

    public static boolean wordBreak(String s, List<String> wordDict) {
        Set<String> dict = new HashSet<>(wordDict);
        int n = s.length();
        boolean[] dp = new boolean[n + 1];
        dp[0] = true;

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < i; j++) {
                if (dp[j] && dict.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[n];
    }

    public static void main(String[] args) {
        System.out.println(wordBreak("leetcode",
            Arrays.asList("leet", "code"))); // true
        System.out.println(wordBreak("catsandog",
            Arrays.asList("cats","dog","sand","and","cat"))); // false
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Greedy (match longest/shortest word first)

    ❌ "catsand" with ["cat","cats","and"] — greedy "cats" leaves "and" ✓, but greedy "cat" leaves "sand" which might not be in dict

    ✅ DP tries ALL possible splits — no greedy shortcut

  2. Not using HashSet for dictionary

    ❌ List.contains() is O(n) per lookup → O(n³) total

    ✅ Convert to HashSet first → O(1) lookup

  3. Not breaking early when dp[i] becomes true

    Not wrong, but wasteful. Once dp[i] = true, no need to check more j values.

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when: ✅ "Can string be segmented into dictionary words?" ✅ "Word segmentation" / "Text tokenization" ✅ DP + dictionary lookup combo

Don't use when: ❌ Need ALL segmentations (use backtracking + DP, LC #140) ❌ Dictionary is a pattern (use regex or Trie)
