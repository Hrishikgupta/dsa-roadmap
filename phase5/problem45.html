#45 — Partition Equal Subset Sum

🔖 Pattern: DP (Subset Sum / 0/1 Knapsack Disguised)

🎯 Real-World Scenario START HERE ▼

Do friends ne milke items kharide: [1, 5, 11, 5] (total = 22). Ab equally split karna hai.

"Kya items ko do groups mein split kar sakte hain with equal total?"

  • [1, 5, 5] = 11 and [11] = 11 → YES ✓
  • For [1, 2, 3, 5] (total=11, odd) → NO ✗ (can't split odd total equally)

This is LeetCode #416 — Partition Equal Subset Sum. 0/1 Knapsack in disguise.

🧠 1. Problem Understanding STEP 1 ▼

The key reduction:

  • Equal partition means each half = totalSum / 2
  • If totalSum is odd → impossible (return false immediately)
  • Problem reduces to: "Can we find a subset with sum = totalSum / 2?"
  • That's the Subset Sum problem — a special case of 0/1 Knapsack!

Knapsack mapping:

  • Items = array elements (each used at most once → 0/1)
  • Weight of each item = its value
  • Capacity = totalSum / 2
  • Question: can we fill the knapsack exactly? (boolean, not max value)

DP recipe:

  1. State: dp[s] = can we make sum s using some subset?
  2. Recurrence: dp[s] = dp[s] || dp[s - nums[i]] (skip or take)
  3. Base: dp[0] = true (empty subset has sum 0)
  4. Order: For each item, iterate s from target DOWN TO nums[i] (0/1 = RIGHT TO LEFT)
  5. Answer: dp[target]
💪 2. Brute Force Thinking STEP 2 ▼

Generate all subsets (#21), check if any has sum = target. O(2ⁿ).

🤔 0/1 Knapsack DP reduces this to O(n × target). Same include/exclude logic, but with boolean dp instead of max value.

⚡ 3. Optimization Thinking STEP 3 ▼

1D boolean DP (same as 0/1 Knapsack #43, but boolean):

boolean[] dp = new boolean[target + 1];
dp[0] = true;

for (int num : nums) {
    for (int s = target; s >= num; s--) { // RIGHT TO LEFT (0/1)
        dp[s] = dp[s] || dp[s - num];
    }
}
return dp[target];

Compare with #43 (0/1 Knapsack):

💡 Subset Sum = 0/1 Knapsack where value = weight and dp is boolean. Exact same structure.

☕ 8. Java Implementation STEP 8 ▼ 
public class PartitionEqualSubset {

    public static boolean canPartition(int[] nums) {
        int total = 0;
        for (int n : nums) total += n;
        if (total % 2 != 0) return false; // odd sum → impossible

        int target = total / 2;
        boolean[] dp = new boolean[target + 1];
        dp[0] = true;

        for (int num : nums) {
            for (int s = target; s >= num; s--) { // RIGHT TO LEFT
                dp[s] = dp[s] || dp[s - num];
            }
            if (dp[target]) return true; // early exit
        }
        return dp[target];
    }

    public static void main(String[] args) {
        System.out.println(canPartition(new int[]{1,5,11,5})); // true
        System.out.println(canPartition(new int[]{1,2,3,5})); // false
    }
}

💡 Early exit: once dp[target] is true, no need to process more elements. Can significantly speed up in practice.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Not checking odd total first

    ✅ if (total % 2 != 0) return false — instant O(1) rejection

  2. Using LEFT TO RIGHT (unbounded style)

    ❌ Allows using same element twice → wrong answer

    ✅ RIGHT TO LEFT — each element used at most once (0/1 knapsack)

  3. Not recognizing this as subset sum

    The hardest part is the reduction: "partition into equal halves" → "find subset with sum = total/2"

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when: ✅ "Can array be split into two equal halves?" ✅ "Does a subset with sum X exist?" ✅ "Target sum with +/- signs"

Don't use when: ❌ Elements can be used multiple times (unbounded) ❌ Target is very large (>10⁶) — O(n×target) too slow
