#46 — Longest Common Subsequence of Two Playlists

🔖 Pattern: DP (Two String DP — LCS)

🎯 Real-World Scenario START HERE ▼

Do friends ke music playlists hain. Kitne songs common hain (in same relative order)?

Playlist A: "abcde"
Playlist B: "ace"

Common subsequence: "ace" (length 3)
  a_c_e  ← from A (skip b, d)
  ace    ← from B (all match)

Question: "Longest common subsequence (LCS) ki length kya hai?"

Answer: 3 ("ace")

This is LeetCode #1143 — Longest Common Subsequence. Foundation for Edit Distance, Diff tools, DNA alignment.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given two strings, find the length of their longest common subsequence
  • Subsequence = characters in order, not necessarily contiguous
  • "ace" is a subsequence of "abcde" (skip b, d)

This is a new DP structure — Two-String DP:

  • Previous DP: 1D (stairs, robber) or items × capacity (knapsack)
  • LCS: dp[i][j] = LCS length of s1[0..i-1] and s2[0..j-1]
  • Two strings = two dimensions

DP recipe:

  1. State: dp[i][j] = LCS of first i chars of s1 and first j chars of s2
  2. Recurrence:
    • If s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1 (match! extend LCS)
    • Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1]) (skip one char from either string)
  3. Base: dp[0][j] = 0, dp[i][0] = 0 (empty string has LCS 0)
  4. Order: Row by row, left to right
  5. Answer: dp[m][n]
💪 2. Brute Force Thinking STEP 2 ▼

Generate all subsequences of s1 (2^m), check each against s2. Exponential.

🤔 At each position, if chars match → both advance + LCS grows. If not → try skipping from s1 OR s2, take the better one. Classic overlapping subproblems.

⚡ 3. Optimization Thinking STEP 3 ▼

The two cases at each (i, j):

if s1[i-1] == s2[j-1]:
    // Characters match! This char is part of LCS.
    dp[i][j] = dp[i-1][j-1] + 1  // diagonal + 1
else:
    // Don't match. Best LCS without one of the chars.
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  // max(top, left)

Intuition:

  • Match: "Great, both strings contributed this character. LCS grows by 1. Move both pointers."
  • No match: "One of these chars isn't in the LCS. Try dropping from s1 (go up) or s2 (go left). Take the better result."

💡 Pattern: Two-String DP — match → diagonal+1, no match → max(top, left). The foundation for Edit Distance, Diff, DNA alignment.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(m × n)
Space:O(m × n)

m = len(s1), n = len(s2). Can optimize space to O(min(m,n)) with rolling array (only need previous row).

☕ 8. Java Implementation STEP 8 ▼ 
public class LongestCommonSubsequence {

    public static int longestCommonSubsequence(String s1, String s2) {
        int m = s1.length(), n = s2.length();
        int[][] dp = new int[m + 1][n + 1];

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (s1.charAt(i - 1) == s2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }
        return dp[m][n];
    }

    public static void main(String[] args) {
        System.out.println(longestCommonSubsequence("abcde", "ace")); // 3
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Confusing subsequence with substring

    Subsequence: skip chars allowed. Substring: must be contiguous. LCS ≠ Longest Common Substring.

  2. Off-by-one with 1-indexed dp

    ✅ dp is (m+1)×(n+1). Access chars with s1.charAt(i-1), not s1.charAt(i).

  3. Using max(top, left, diagonal) for no-match

    ❌ Diagonal is always ≤ max(top, left) for LCS. Including it is redundant but not wrong — just unnecessary.

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when: ✅ "Longest common subsequence" ✅ "Minimum deletions to make strings equal" (= m+n - 2×LCS) ✅ "Shortest common supersequence" ✅ Any two-string comparison problem

Don't use when: ❌ "Longest common substring" (contiguous — different recurrence, reset to 0 on mismatch) ❌ Single string problems (use 1D DP)
