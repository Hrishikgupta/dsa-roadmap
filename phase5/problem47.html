#47 — Edit Distance — Autocorrect Engine

🔖 Pattern: DP (Edit Distance / Levenshtein)

🎯 Real-World Scenario START HERE ▼

Tum ek autocorrect engine bana rahe ho. User ne type kiya "horse", dictionary mein closest word hai "ros".

"Minimum kitne edits lagenge 'horse' ko 'ros' mein convert karne mein?"

  • horse → rorse (replace h with r) → 1 edit
  • rorse → rose (replace r with s, wait — let me redo)
horse → rorse  (replace 'h' → 'r')
rorse → rose   (delete 'r' at index 2)
rose  → ros    (delete 'e')
= 3 operations

This is LeetCode #72 — Edit Distance (Levenshtein Distance). Used in spell checkers, DNA alignment, NLP.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given two strings, find minimum number of operations to convert s1 into s2
  • Three operations allowed: Insert, Delete, Replace (each costs 1)

Builds directly on LCS (#46):

Aspect #46 LCS #47 Edit Distance
GoalMaximize commonMinimize operations
Matchdiag + 1diag + 0 (no cost)
No matchmax(top, left)1 + min(top, left, diag)
Basedp[0][j]=0, dp[i][0]=0dp[0][j]=j, dp[i][0]=i

DP recipe:

  1. State: dp[i][j] = min edits to convert s1[0..i-1] to s2[0..j-1]
  2. Recurrence:
    • Match: dp[i][j] = dp[i-1][j-1] (no operation needed)
    • No match: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
  3. Base: dp[i][0] = i (delete all), dp[0][j] = j (insert all)
  4. Answer: dp[m][n]

The three operations mapped to dp transitions:

  • dp[i-1][j] + 1 = Delete from s1 (remove s1[i-1], match remaining)
  • dp[i][j-1] + 1 = Insert into s1 (insert s2[j-1], match remaining)
  • dp[i-1][j-1] + 1 = Replace s1[i-1] with s2[j-1]
⚡ 3. Optimization Thinking STEP 3 ▼ 
if s1[i-1] == s2[j-1]:
    dp[i][j] = dp[i-1][j-1]          // match — free!
else:
    dp[i][j] = 1 + min(
        dp[i-1][j],      // delete from s1
        dp[i][j-1],      // insert into s1
        dp[i-1][j-1]     // replace in s1
    )

💡 Same 2D table as LCS. Match = free (diagonal, no cost). No match = 1 + min of three neighbors (delete, insert, replace).

⚙️ 6. Internal Mechanics STEP 6 ▼

Relationship between LCS and Edit Distance:

editDistance(s1, s2) = m + n - 2 × LCS(s1, s2)

(When only insert and delete are allowed, no replace. With replace, the formula is different.)

Real-world applications:

  • Spell checkers — suggest words with smallest edit distance
  • Autocorrect — "teh" → "the" (1 swap ≈ 1 edit)
  • DNA alignment — mutations = edits between gene sequences
  • Plagiarism detection — how different are two texts?
  • Fuzzy search — find approximate matches in databases
📊 7. Complexity Analysis STEP 7 ▼
Time:O(m × n)
Space:O(m × n)

Can optimize space to O(min(m,n)) with rolling array.

☕ 8. Java Implementation STEP 8 ▼ 
public class EditDistance {

    public static int minDistance(String s1, String s2) {
        int m = s1.length(), n = s2.length();
        int[][] dp = new int[m + 1][n + 1];

        for (int i = 0; i <= m; i++) dp[i][0] = i;
        for (int j = 0; j <= n; j++) dp[0][j] = j;

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (s1.charAt(i-1) == s2.charAt(j-1)) {
                    dp[i][j] = dp[i-1][j-1];
                } else {
                    dp[i][j] = 1 + Math.min(dp[i-1][j],
                                   Math.min(dp[i][j-1], dp[i-1][j-1]));
                }
            }
        }
        return dp[m][n];
    }

    public static void main(String[] args) {
        System.out.println(minDistance("horse", "ros"));       // 3
        System.out.println(minDistance("intention", "execution")); // 5
    }
}

💡 Almost identical to LCS (#46). Base cases change (i and j instead of 0), and no-match uses min of THREE neighbors + 1 instead of max of two.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Base cases: dp[i][0] = 0 instead of i

    ❌ Converting "abc" to "" needs 3 deletions, not 0

    ✅ dp[i][0] = i (delete i chars), dp[0][j] = j (insert j chars)

  2. Forgetting the diagonal (replace) option on mismatch

    ❌ Only considering insert and delete → suboptimal

    ✅ min of ALL THREE: top (delete), left (insert), diagonal (replace)

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when: ✅ "Minimum edits to transform string A to B" ✅ "Similarity between two strings" ✅ Spell checking, fuzzy matching, DNA alignment

Don't use when: ❌ Only need to know if strings are equal (just compare directly) ❌ Operations have different costs (need weighted edit distance — same structure, different costs)

💡 "Edit Distance = LCS's twin. Same 2D table. Match → diagonal (free). No match → 1 + min(delete=top, insert=left, replace=diagonal). Base: dp[i][0]=i, dp[0][j]=j."
