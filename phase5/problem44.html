#44 — Coin Change — Minimum Notes for Bill

🔖 Pattern: DP (Unbounded Knapsack)

🎯 Real-World Scenario START HERE ▼

ATM mein coins hain: [1, 5, 10, 25] cents. Customer ko 30 cents dene hain.

"Minimum kitne coins lagenge?"

  • 25 + 5 = 30 → 2 coins ✓ (optimal)
  • 10 + 10 + 10 = 30 → 3 coins
  • 25 + 1 + 1 + 1 + 1 + 1 = 30 → 6 coins

Answer: 2 coins (25 + 5)

This is LeetCode #322 — Coin Change. Classic Unbounded Knapsack (unlimited copies of each coin).

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given coin denominations and a target amount
  • Find minimum number of coins to make that amount
  • Unlimited supply of each denomination
  • Return -1 if impossible

0/1 Knapsack (#43) vs Unbounded Knapsack (#44):

Aspect #43 (0/1 Knapsack) #44 (Unbounded)
Item usageAt most onceUnlimited copies
1D loop directionRIGHT TO LEFTLEFT TO RIGHT
Optimizemax valuemin coins (this problem)
ExamplePacking trip bagMaking change

DP recipe:

  1. State: dp[amount] = minimum coins to make this amount
  2. Recurrence: dp[a] = min(dp[a], dp[a - coin] + 1) for each coin
  3. Base: dp[0] = 0 (0 coins for amount 0)
  4. Order: Amount 1 to target, LEFT TO RIGHT (allows reuse)
  5. Answer: dp[amount] (or -1 if still ∞)
💪 2. Brute Force Thinking STEP 2 ▼

Greedy (take largest coin first) — WRONG for some cases:

  • Coins [1, 3, 4], amount = 6
  • Greedy: 4 + 1 + 1 = 3 coins
  • Optimal: 3 + 3 = 2 coins

Greedy fails because taking the largest coin isn't always optimal. DP considers all possibilities.

🤔 For each amount, try using each coin and take the minimum. dp[6] = min(dp[6-1]+1, dp[6-3]+1, dp[6-4]+1)

⚡ 3. Optimization Thinking STEP 3 ▼

Bottom-up DP:

dp[0] = 0;  // base: 0 coins for amount 0
dp[1..amount] = ∞;  // initially impossible

for (int a = 1; a <= amount; a++) {
    for (int coin : coins) {
        if (coin <= a && dp[a - coin] != ∞) {
            dp[a] = Math.min(dp[a], dp[a - coin] + 1);
        }
    }
}

Why LEFT TO RIGHT works for unlimited coins:

  • When computing dp[6] using coin 3: dp[6] = dp[6-3] + 1 = dp[3] + 1
  • dp[3] was already computed (left to right) and might already use coin 3
  • So dp[6] = dp[3] + 1 = (use coin 3 for dp[3]) + 1 = two coin-3's. Reuse allowed!
  • In 0/1 knapsack, right-to-left prevents this reuse. Here we WANT it.

💡 Pattern: Unbounded Knapsack — same as 0/1 but LEFT TO RIGHT (allows reuse). For min coins: use min instead of max.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.Arrays;

public class CoinChange {

    public static int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1); // "infinity"
        dp[0] = 0;

        for (int a = 1; a <= amount; a++) {
            for (int coin : coins) {
                if (coin <= a) {
                    dp[a] = Math.min(dp[a], dp[a - coin] + 1);
                }
            }
        }

        return dp[amount] > amount ? -1 : dp[amount];
    }

    public static void main(String[] args) {
        System.out.println(coinChange(new int[]{1,3,4}, 6));  // 2
        System.out.println(coinChange(new int[]{2}, 3));      // -1
    }
}

💡 Using amount + 1 as "infinity" instead of Integer.MAX_VALUE — avoids overflow when adding +1. If dp[amount] > amount → impossible (can't need more than 'amount' coins of denomination 1).

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Using greedy (largest coin first)

    ❌ Fails for [1,3,4] amount=6. Greedy: 4+1+1=3. Optimal: 3+3=2.

    ✅ Always use DP for coin change

  2. Using Integer.MAX_VALUE as infinity

    ❌ MAX_VALUE + 1 overflows to negative → wrong min comparison

    ✅ Use amount + 1 as infinity — safe and sufficient

  3. Iterating RIGHT TO LEFT (0/1 knapsack style)

    ❌ Prevents coin reuse → each coin used at most once → wrong answer

    ✅ LEFT TO RIGHT for unlimited coins

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use when:

  • ✅ "Minimum coins/items to reach target" with unlimited supply
  • ✅ "Number of ways to make change"
  • ✅ Any unbounded knapsack variant

Don't use when:

  • ❌ Each item used at most once → 0/1 Knapsack (#43)
  • ❌ Greedy works (e.g., US coin system [1,5,10,25] — greedy happens to be optimal)
💡 "Coin Change = Unbounded Knapsack with min. dp[a] = min(dp[a-coin] + 1) for each coin. LEFT TO RIGHT allows reuse. Greedy fails — always use DP."
