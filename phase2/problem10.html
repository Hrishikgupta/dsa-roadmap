#10 — Longest Unique Song Playlist

🔖 Pattern: Sliding Window + HashMap

🎯 Real-World Scenario START HERE ▼

Tum ek music streaming app bana rahe ho. User ki listening history hai:

"abcabcbb" (each character = a song ID)

Feature request: "Show the longest streak where the user didn't repeat any song."

Possible streaks:

  • "abc" = 3 unique songs ✓
  • "bca" = 3 unique songs ✓
  • "cab" = 3 unique songs ✓
  • "abcabcbb" full = has repeats ✗

Answer: 3 (longest substring without repeating characters)

This is LeetCode #3 — one of the most asked interview questions across all companies.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a string, find the length of the longest substring without repeating characters
  • Substring = contiguous (not subsequence)

Input → Output:

  • Input: String s
  • Output: int — length of longest unique-character substring

This problem combines two patterns we already know:

From Problem Pattern Used Here For
#2HashMapTrack which characters are in the window + their last position

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Check every substring, verify all characters are unique.

// Brute Force — O(n³)
int maxLen = 0;
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        if (allUnique(s, i, j)) {  // O(n) check
            maxLen = Math.max(maxLen, j - i + 1);
        }
    }
}

O(n²) substrings × O(n) uniqueness check = O(n³). Terrible.

Slightly better: Use HashSet per starting position — O(n²).

🤔 Problem #9 mein humne seekha: variable window expand/shrink karo. Yahan condition hai "all unique". Jab repeat aaye → shrink. Jab unique ho → expand.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — Two approaches, both O(n):

Approach A: Sliding Window + HashSet (shrink one-by-one)

  • Expand right: add character to set
  • If character already in set → shrink left one-by-one, removing from set, until duplicate is gone
  • Track max window size

Approach B: Sliding Window + HashMap (jump left directly) ⭐

  • HashMap stores each character's last seen index
  • When a repeat is found at position right:
  • Instead of shrinking one-by-one, jump left directly to map.get(char) + 1
  • Faster in practice — avoids unnecessary shrink steps

Why HashMap is better than HashSet here:

  • HashSet: when 'a' repeats, you shrink left one-by-one until 'a' is removed. Could be many steps.
  • HashMap: you know exactly WHERE 'a' was last seen → jump left there directly. One step.

💡 Pattern: Variable Sliding Window + HashMap (last-seen index) — for longest unique substring

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Variable Sliding Window + HashMap (last-seen index)

Algorithm:

  1. left = 0, maxLen = 0, HashMap<Character, Integer> map
  2. For right = 0 to n - 1:
    • If s.charAt(right) is in map AND its last index ≥ left:
      • left = map.get(char) + 1 (jump past the duplicate)
    • map.put(char, right) (update last-seen position)
    • maxLen = Math.max(maxLen, right - left + 1)
  3. Return maxLen

💡 The check map.get(char) >= left is crucial — if the duplicate is BEFORE our current window, we ignore it.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why map.get(char) >= left is critical:

Consider: s = "abba"

  • R=0: a→0, L=0, window="a"
  • R=1: b→1, L=0, window="ab"
  • R=2: b repeats at 1, 1≥0 → L=2, b→2, window="b"
  • R=3: a is in map at 0, but 0 < L(2) → a is OUTSIDE our window → ignore it!

Without the >= left check, we'd wrongly jump left back to 1, which is before our current window. The map has stale entries from before the window — we must ignore them.

HashSet approach vs HashMap approach:

Aspect HashSet HashMap (last index)
On duplicateShrink left one-by-oneJump left directly
Worst caseO(n) — sameO(n) — same
Practical speedSlower (many shrink steps)Faster (one jump)
Code complexitySimplerSlightly trickier (≥ left check)
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n)
Space: O(min(n, m))

Where m = size of character set (26 for lowercase, 128 for ASCII, 65536 for Unicode).

HashMap stores at most m entries. For ASCII strings, space is effectively O(1) = O(128).

☕ 8. Java Implementation STEP 8 ▼

Approach 1: HashMap (optimal — jump left)

import java.util.HashMap;

public class LongestUniqueSubstring {

    public static int lengthOfLongestSubstring(String s) {
        HashMap<Character, Integer> map = new HashMap<>();
        int left = 0, maxLen = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);

            if (map.containsKey(c) && map.get(c) >= left) {
                left = map.get(c) + 1;
            }

            map.put(c, right);
            maxLen = Math.max(maxLen, right - left + 1);
        }

        return maxLen;
    }

    public static void main(String[] args) {
        System.out.println(lengthOfLongestSubstring("abcabcbb")); // 3
        System.out.println(lengthOfLongestSubstring("bbbbb"));    // 1
        System.out.println(lengthOfLongestSubstring("pwwkew"));   // 3
    }
}

Approach 2: int[128] array (fastest for ASCII)

public static int lengthOfLongestSubstringFast(String s) {
    int[] lastSeen = new int[128];
    Arrays.fill(lastSeen, -1);
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastSeen[c] >= left) {
            left = lastSeen[c] + 1;
        }
        lastSeen[c] = right;
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}

💡 Array lookup is faster than HashMap. Use int[128] for ASCII, int[256] for extended ASCII. Initialize with -1 (never seen).

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Missing the >= left check

    ❌ if (map.containsKey(c)) left = map.get(c) + 1

    Fails on "abba" — jumps left backwards when 'a' is found at index 0 (before current window)

    ✅ if (map.containsKey(c) && map.get(c) >= left)

  2. Not updating map for every character

    ❌ Only updating map when character is new

    ✅ Always map.put(c, right) — update last-seen position even for repeats

  3. Confusing substring with subsequence

    Substring = contiguous. "ace" is a subsequence of "abcde" but not a substring.

    Sliding window works for substrings. Subsequences need different approaches (DP).

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify) this pattern:

  • ✅ "Longest substring without repeating characters"
  • ✅ "Longest substring with at most K distinct characters"
  • ✅ "Longest subarray with at most K different elements"
  • ✅ Any "longest/shortest contiguous" + "uniqueness/frequency constraint"

The Sliding Window + HashMap family:

  • This problem: HashMap tracks last-seen index
  • K distinct chars: HashMap tracks frequency count
  • Min window substring (#12): HashMap tracks required vs found counts
  • All use the same expand/shrink framework — just different window state
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Longest subsequence" (not substring) — needs DP, not sliding window
  • ❌ Non-contiguous selection — sliding window only works on contiguous ranges
  • ❌ When the "uniqueness" condition involves complex relationships between elements (not just presence/absence)
💡 "Sliding Window + HashMap = power combo. Window handles the range, HashMap handles the state. Together they solve most substring problems in O(n)."
