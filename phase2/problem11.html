#11 — Checking if Two Words are Anagram Passwords

🔖 Pattern: Frequency Map Comparison

🎯 Real-World Scenario START HERE ▼

Tumhari company mein ek password security rule hai:

"New password must be an anagram of the old password."

Same characters, different arrangement. Taaki user ko yaad rahe but hackers ko guess na ho.

  • Old: "listen" → New: "silent" → ✅ Anagram
  • Old: "hello" → New: "world" → ❌ Not anagram

Question: Given two strings, check if they are anagrams.

This is LeetCode #242 — Valid Anagram.

🧠 1. Problem Understanding STEP 1 ▼

Two strings are anagrams if they contain the exact same characters with the exact same frequencies.

  • "listen" → {l:1, i:1, s:1, t:1, e:1, n:1}
  • "silent" → {s:1, i:1, l:1, e:1, n:1, t:1} → Same! ✓

Connection to previous:

  • #2: HashMap for frequency counting — same technique, now comparing TWO maps
  • #10: HashMap tracked last-seen index — here we track character counts
  • This sets up #12 (Min Window Substring) where frequency comparison happens inside a sliding window
💪 2. Brute Force Thinking STEP 2 ▼

Sort both strings, compare — O(n log n):

char[] a = s.toCharArray(), b = t.toCharArray();
Arrays.sort(a); Arrays.sort(b);
return Arrays.equals(a, b);

Works but O(n log n). Can we do O(n)?

🤔 Anagram = same frequency. Frequency counting is O(n) with int[26]. Count up for one string, count down for the other. All zeros? Anagram.

⚡ 3. Optimization Thinking STEP 3 ▼

Single Counter Technique:

  • For each char in s: count[char - 'a']++
  • For each char in t: count[char - 'a']--
  • If all counts are 0 → anagram. Any count ≠ 0 → not anagram.

Think of it as a balance sheet: s deposits characters, t withdraws them. Balance = 0 for all → anagram.

int[26] vs HashMap:

Aspect int[26] HashMap
Character setOnly lowercase a-zAny character (Unicode)
SpeedFaster (array access)Slower (hashing)
SpaceO(1) — fixed 26O(k) — k unique chars

💡 Pattern: Frequency Map Comparison — count up for one, count down for the other, check all zeros.

Time:O(n)
Space:O(1)

One pass through both strings + one pass through count array (26 = constant).

☕ 8. Java Implementation STEP 8 ▼

int[26] — fastest (lowercase only):

public class ValidAnagram {

    public static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;

        int[] count = new int[26];

        for (int i = 0; i < s.length(); i++) {
            count[s.charAt(i) - 'a']++;
            count[t.charAt(i) - 'a']--;
        }

        for (int c : count) {
            if (c != 0) return false;
        }
        return true;
    }

    public static void main(String[] args) {
        System.out.println(isAnagram("anagram", "nagaram")); // true
        System.out.println(isAnagram("rat", "car"));         // false
    }
}

HashMap version (Unicode-safe):

public static boolean isAnagramUnicode(String s, String t) {
    if (s.length() != t.length()) return false;
    HashMap<Character, Integer> map = new HashMap<>();
    for (char c : s.toCharArray())
        map.put(c, map.getOrDefault(c, 0) + 1);
    for (char c : t.toCharArray()) {
        int val = map.getOrDefault(c, 0) - 1;
        if (val < 0) return false;
        map.put(c, val);
    }
    return true;
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting the length check

    ✅ Always check s.length() != t.length() first — O(1) early exit

  2. Using int[26] for non-lowercase input

    ✅ Ask interviewer about charset. Use int[128] for ASCII, HashMap for Unicode.

  3. Case sensitivity

    If case-insensitive: s.toLowerCase() first.
