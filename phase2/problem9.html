#9 — Shortest Refueling Stop on Highway

🔖 Pattern: Sliding Window (Variable Size)

🎯 Real-World Scenario START HERE ▼

Tum ek highway pe drive kar rahe ho. Har kilometer pe ek toll booth hai jahan kuch fuel milta hai:

[2, 3, 1, 2, 4, 3] (litres of fuel at each stop)

Tumhe minimum 7 litres fuel collect karna hai to reach the destination.

Question: Kam se kam kitne consecutive stops pe rukna padega to collect ≥ 7 litres?

Possible answers:

  • [2,3,1,2] = 8 ≥ 7 → length 4
  • [3,1,2,4] = 10 ≥ 7 → length 4
  • [4,3] = 7 ≥ 7 → length 2 ✓ (shortest!)

💡 Window size fixed nahi hai — it depends on when the sum condition is met. Expand karo jab tak condition na mile, shrink karo jab mil jaaye.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an array of positive integers and a target sum
  • Find the minimum length subarray whose sum is ≥ target
  • If no such subarray exists, return 0

Input → Output:

  • Input: int target, int[] nums (all positive)
  • Output: int — minimum length (or 0 if impossible)

This is LeetCode #209 — Minimum Size Subarray Sum

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Try every possible subarray, check if sum ≥ target, track minimum length.

// Brute Force — O(n²)
int minLen = Integer.MAX_VALUE;
for (int i = 0; i < n; i++) {
    int sum = 0;
    for (int j = i; j < n; j++) {
        sum += nums[j];
        if (sum >= target) {
            minLen = Math.min(minLen, j - i + 1);
            break; // no need to extend further
        }
    }
}
return minLen == Integer.MAX_VALUE ? 0 : minLen;

O(n²) — the break helps but worst case is still quadratic.

🤔 Jab hum inner loop mein sum ≥ target paate hain aur break karte hain, toh next iteration mein phir se scratch se start karte hain. Kya hum pichle kaam ko reuse kar sakte hain?

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Caterpillar Analogy:

Imagine a caterpillar on the array. It has a head (right) and tail (left).

  • Hungry? (sum < target) → Stretch head forward (expand right) to eat more
  • Full? (sum ≥ target) → Pull tail forward (shrink left) to see if you can be shorter while still full

Why this works with positive numbers:

  • All elements are positive → expanding always increases sum, shrinking always decreases sum
  • So the window behaves predictably — no surprises
  • Left pointer never goes back, right pointer never goes back → each element visited at most twice → O(n)

The two-phase loop:

// Right pointer expands (outer loop)
for (right = 0; right < n; right++) {
    windowSum += nums[right];  // expand

    // Left pointer shrinks (inner while)
    while (windowSum >= target) {
        update answer;
        windowSum -= nums[left]; // shrink
        left++;
    }
}

💡 Pattern identified: Variable-Size Sliding Window — expand right, shrink left when condition met

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern Name: Sliding Window — Variable Size (Shrinkable)

Algorithm:

  1. left = 0, windowSum = 0, minLen = Integer.MAX_VALUE
  2. For right = 0 to n - 1:
    • windowSum += nums[right] (expand window)
    • While windowSum >= target:
      • minLen = Math.min(minLen, right - left + 1)
      • windowSum -= nums[left] (shrink from left)
      • left++
  3. Return minLen == MAX_VALUE ? 0 : minLen

💡 The while loop inside the for loop looks like O(n²), but it's not — left moves at most n times total across ALL iterations of the outer loop.

Left moved 5 times total, right moved 6 times. Total work ≈ 11 = O(n), not O(n²).

⚙️ 6. Internal Mechanics STEP 6 ▼

Why it's O(n) despite the nested while loop:

  • The right pointer moves from 0 to n-1 → n moves total
  • The left pointer also moves from 0 to at most n-1 → n moves total
  • Left never goes back, right never goes back
  • Total pointer movements = n + n = 2n = O(n)

This is called "amortized O(n)" — the inner while loop runs many times in some iterations but zero times in others. Total across all iterations = O(n).

Why this only works with positive numbers:

  • Positive numbers guarantee: expanding → sum increases, shrinking → sum decreases
  • This monotonic behavior lets us confidently shrink when sum ≥ target
  • With negative numbers, shrinking could INCREASE the sum → the greedy shrink logic breaks
  • For arrays with negatives, you'd need prefix sum + binary search or deque-based approaches

☕ 8. Java Implementation STEP 8 ▼ 
public class MinSubarraySum {

    public static int minSubArrayLen(int target, int[] nums) {
        int left = 0;
        int windowSum = 0;
        int minLen = Integer.MAX_VALUE;

        for (int right = 0; right < nums.length; right++) {
            windowSum += nums[right];

            while (windowSum >= target) {
                minLen = Math.min(minLen, right - left + 1);
                windowSum -= nums[left];
                left++;
            }
        }

        return minLen == Integer.MAX_VALUE ? 0 : minLen;
    }

    public static void main(String[] args) {
        int[] fuel = {2, 3, 1, 2, 4, 3};
        System.out.println(minSubArrayLen(7, fuel)); // 2
    }
}

💡 12 lines of logic. The beauty of sliding window — simple code, powerful result.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Using if instead of while for shrinking

    ❌ if (windowSum >= target) — only shrinks once, might miss shorter windows

    ✅ while (windowSum >= target) — keeps shrinking until condition breaks

  2. Updating answer AFTER shrinking

    ❌ Shrink first, then record length → misses the valid window before shrink

    ✅ Record length first, THEN shrink

  3. Applying this to arrays with negative numbers

    ❌ Shrinking doesn't guarantee sum decreases with negatives

    For negatives, use prefix sum + sorted structure or deque

  4. Returning minLen without checking if it was updated

    ❌ Returns Integer.MAX_VALUE when no valid subarray exists

    ✅ return minLen == MAX_VALUE ? 0 : minLen

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify) this pattern:

  • ✅ "Shortest/smallest subarray with sum ≥ X"
  • ✅ "Longest subarray with sum ≤ X" (flip the shrink condition)
  • ✅ "Minimum window containing all characters" (Problem #12)
  • ✅ "Longest substring without repeating characters" (Problem #10)
  • ✅ Any problem asking for shortest/longest contiguous subarray meeting a condition

Variable window template:

int left = 0;
for (int right = 0; right < n; right++) {
    // 1. Expand: add nums[right] to window state

    while (/* window is valid / condition met */) {
        // 2. Update answer (for shortest)
        // 3. Shrink: remove nums[left] from window state
        left++;
    }
    // For longest: update answer here (window just became invalid)
}
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Array contains negative numbers (sum isn't monotonic with window size)
  • ❌ Non-contiguous elements needed ("pick any k elements" → sorting/heap)
  • ❌ Window size is fixed → use fixed sliding window (#8), simpler
  • ❌ "Maximum subarray sum" (any size) → Kadane's algorithm, not sliding window
💡 "Variable window = caterpillar. Hungry? Expand right. Full? Shrink left. Both pointers only move forward → O(n)."
