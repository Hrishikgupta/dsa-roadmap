#7 — Triplet Combo for Gift Packing

🔖 Pattern: Two Pointer (3Sum)

🎯 Real-World Scenario START HERE ▼

Tum ek gift shop mein ho. Items ke weights hain:

[-4, -1, -1, 0, 1, 2]

Boss bola: "Find all unique triplets (3 items) whose weights add up to exactly 0."

Yeh balance check hai — positive aur negative weights cancel hone chahiye.

Answer: [[-1, -1, 2], [-1, 0, 1]]

⚠️ Catch: Duplicate triplets not allowed. [-1, -1, 2] sirf ek baar aana chahiye, chahe -1 do baar ho array mein.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an array, find ALL unique triplets (a, b, c) such that a + b + c = 0
  • No duplicate triplets in the result
  • Order within triplet doesn't matter

Input → Output:

  • Input: int[] nums
  • Output: List<List<Integer>> — all unique triplets

Key difference from Problem #6:

Aspect Problem #6 (Two Sum) Problem #7 (3Sum)
Numbers needed23
ResultsOne pairALL unique triplets
DuplicatesNot an issueMust handle carefully
TargetGivenFixed at 0 (usually)

Core idea: 3Sum = Fix one number + Two Sum on the rest. Reduce 3-number problem to 2-number problem.

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Three nested loops — check every triplet.

// Brute Force — O(n³)
Set<List<Integer>> result = new HashSet<>();
for (int i = 0; i < n; i++)
  for (int j = i+1; j < n; j++)
    for (int k = j+1; k < n; k++)
      if (nums[i]+nums[j]+nums[k] == 0) {
        List<Integer> tri = Arrays.asList(nums[i],nums[j],nums[k]);
        Collections.sort(tri);
        result.add(tri);
      }

Problems:

  • O(n³) — for n = 3000, that's 27 billion operations. Way too slow.
  • Using Set for dedup is messy and adds overhead

🤔 Agar hum ek number fix karein, toh baaki do ke liye Problem #6 ka two pointer laga sakte hain — O(n²) total.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — Reduce to a solved problem:

  • a + b + c = 0 → b + c = -a
  • Fix a = nums[i], then find pair (b, c) with sum = -nums[i]
  • That's exactly Problem #6 (Two Sum on sorted array)!

Algorithm sketch:

  1. Sort the array
  2. For each i from 0 to n-3:
    • Fix nums[i] as the first number
    • Use two pointers (left = i+1, right = n-1) to find pair summing to -nums[i]

The hard part — handling duplicates:

  • Array: [-1, -1, 0, 1] → both -1's would produce the same triplet [-1, 0, 1]
  • Solution: Skip duplicate values at two levels:
    • Outer loop: if nums[i] == nums[i-1], skip (same first element)
    • Inner loop: after finding a pair, skip duplicate left and right values

💡 Pattern: Sort + Fix one + Two Pointer for remaining two + Skip duplicates

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern Name: Sort + Fix One + Two Pointer (3Sum)

Complete Algorithm:

  1. Sort the array
  2. For i = 0 to n - 3:
    • Optimization: if nums[i] > 0, break (sorted — no way three positives sum to 0)
    • Skip duplicate: if i > 0 && nums[i] == nums[i-1], continue
    • Set left = i + 1, right = n - 1
    • While left < right:
      • sum = nums[i] + nums[left] + nums[right]
      • If sum == 0: add triplet, skip duplicate lefts, skip duplicate rights
      • If sum < 0: left++
      • If sum > 0: right--

⚙️ 6. Internal Mechanics STEP 6 ▼

How duplicate skipping works (the tricky part):

Level 1 — Outer loop skip:

if (i > 0 && nums[i] == nums[i - 1]) continue;
  • If current first element is same as previous, all triplets starting with it were already found
  • Example: [-1, -1, 0, 1] — second -1 would produce same triplets as first -1

Level 2 — Inner loop skip (after finding a triplet):

// After adding triplet to result:
while (left < right && nums[left] == nums[left + 1]) left++;
while (left < right && nums[right] == nums[right - 1]) right--;
left++;
right--;
  • After finding [-1, -1, 2], if there's another -1 next to left, skip it
  • Same for right — skip duplicate values
  • Then move both pointers inward to look for new pairs

Why nums[i] > 0 means we can break:

  • Array is sorted. If nums[i] > 0, then nums[left] and nums[right] are also > 0
  • Three positive numbers can never sum to 0
  • Early termination — saves unnecessary iterations
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n²)
Space: O(1)*

Breakdown:

  • Sorting: O(n log n)
  • Outer loop: O(n) iterations
  • Inner two-pointer: O(n) per iteration
  • Total: O(n log n) + O(n × n) = O(n²)

*Space O(1) ignoring the output list. Sorting is in-place. Some argue O(log n) for sort's stack space.

Approach Time n = 3000
Brute ForceO(n³)27,000,000,000
Sort + Two PointerO(n²)9,000,000

3000x faster. O(n²) is the best known for 3Sum — no O(n log n) solution exists.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class ThreeSum {

    public static List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);
        int n = nums.length;

        for (int i = 0; i < n - 2; i++) {
            // Early termination
            if (nums[i] > 0) break;

            // Skip duplicate first element
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            int left = i + 1, right = n - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // Skip duplicates
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;

                    left++;
                    right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return result;
    }

    public static void main(String[] args) {
        int[] nums = {-1, 0, 1, 2, -1, -4};
        System.out.println(threeSum(nums));
        // [[-1, -1, 2], [-1, 0, 1]]
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting to sort first

    ❌ Two pointer only works on sorted arrays. Without sort, results are wrong.

  2. Wrong duplicate skip condition

    ❌ if (nums[i] == nums[i+1]) continue — skips valid first occurrences

    ✅ if (i > 0 && nums[i] == nums[i-1]) continue — skips only repeats

  3. Not moving both pointers after finding a triplet

    ❌ Only doing left++ or only right-- → infinite loop or missed results

    ✅ After skipping duplicates, move BOTH left++ and right--

  4. Using HashSet for dedup instead of skip logic

    Works but adds O(n) space and is slower. The skip approach is cleaner and O(1) space.

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify) this pattern:

  • ✅ "Find triplets with sum = X"
  • ✅ "3Sum" / "Three number sum"
  • ✅ "Find all unique combinations of k numbers" (generalize to kSum)
  • ✅ Any problem reducible to "fix one, two-pointer the rest"

The kSum generalization:

// Pattern for any kSum:
2Sum → Two Pointer                    O(n)
3Sum → Fix 1 + Two Pointer            O(n²)
4Sum → Fix 1 + Fix 1 + Two Pointer    O(n³)
kSum → Fix (k-2) + Two Pointer        O(n^(k-1))
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ When you need indices from the original unsorted array (sorting destroys indices)
  • ❌ When k is very large (kSum with k=10 → O(n⁹) — need different approach)
  • ❌ When the target changes per query (pre-processing might help)
  • ❌ "Closest sum to target" — similar approach but comparison logic changes (3Sum Closest — LC #16)
💡 "3Sum = Fix one + Two Sum. kSum = Fix (k-2) + Two Sum. Reduce to what you already know."
