#14 — Container With Most Water in a Tank

🔖 Pattern: Two Pointer (Greedy Shrink)

🎯 Real-World Scenario START HERE ▼

This is LeetCode #11 — Container With Most Water. Classic two-pointer problem.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given n vertical lines, find two lines that together with the x-axis form a container holding the most water
  • Area = min(height[i], height[j]) × (j - i)
  • Water level = shorter wall (water overflows over the shorter one)
  • Width = distance between the two walls

Input → Output:

  • Input: int[] height
  • Output: int — maximum area

#13 (Trap Water) vs #14 (Container) — they look similar but are VERY different:

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Try every pair of walls.

// Brute Force — O(n²)
int maxArea = 0;
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        int area = Math.min(height[i], height[j]) * (j - i);
        maxArea = Math.max(maxArea, area);
    }
}

O(n²) pairs. For n = 100,000 → 5 billion. TLE.

🤔 Area = width × height. Start with maximum width (left=0, right=n-1). As we shrink width, we need taller walls to compensate. Which wall should we move?

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Greedy Argument:

Start with the widest container (left=0, right=n-1). Now we must shrink.

  • Area = min(h[L], h[R]) × width
  • Width will decrease by 1 no matter what. So we need height to increase.
  • Height is limited by the shorter wall.
  • Moving the taller wall inward: width decreases, height can only stay same or decrease → area CANNOT increase
  • Moving the shorter wall inward: width decreases, but height MIGHT increase → area COULD increase

So the greedy choice is clear: always move the shorter wall.

Why we never miss the optimal:

  • When we move the shorter wall, we're discarding all pairs involving that wall with any inner position
  • But all those pairs have the same or smaller width AND the same bottleneck height → they can't be better
  • So it's safe to discard them

💡 Pattern: Two Pointer (Greedy Shrink) — start widest, move the shorter wall inward

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Two Pointer — Greedy Shrink from Opposite Ends

Algorithm:

  1. left = 0, right = n - 1, maxArea = 0
  2. While left < right:
    • area = min(height[left], height[right]) × (right - left)
    • maxArea = max(maxArea, area)
    • If height[left] < height[right]: left++
    • Else: right--
  3. Return maxArea

That's it. The simplest code among all Phase 2 problems — but the greedy proof is the hard part.

⚙️ 6. Internal Mechanics STEP 6 ▼

Formal proof — why moving the shorter wall is optimal:

Suppose h[L] < h[R]. Consider all containers involving wall L:

  • Container (L, R): area = h[L] × (R-L)
  • Container (L, R-1): area = min(h[L], h[R-1]) × (R-L-1)
  • Container (L, R-2): area = min(h[L], h[R-2]) × (R-L-2)
  • ...

For ALL of these:

  • Width is strictly less than (R-L)
  • Height is at most h[L] (since h[L] is the shorter wall, min is capped at h[L])
  • So area ≤ h[L] × (R-L) = area we already computed

Therefore, no container involving wall L can be better than (L, R). Safe to discard L and move to L+1.

What if heights are equal?

  • If h[L] == h[R], moving either pointer is fine
  • Both sides are equally "bottlenecked" — neither can produce a better result with the current partner
  • Convention: move either one (most implementations move right, some move left)
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n)
Space: O(1)

Each step moves one pointer. Total steps = n-1. Constant work per step.

☕ 8. Java Implementation STEP 8 ▼ 
public class ContainerWithMostWater {

    public static int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int maxArea = 0;

        while (left < right) {
            int area = Math.min(height[left], height[right]) * (right - left);
            maxArea = Math.max(maxArea, area);

            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        return maxArea;
    }

    public static void main(String[] args) {
        int[] h = {1,8,6,2,5,4,8,3,7};
        System.out.println(maxArea(h)); // 49
    }
}

💡 Possibly the cleanest code in the entire roadmap. 10 lines of logic. The elegance of two pointers.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Confusing with Trapping Rain Water (#13)

    ❌ Trying to sum water at each position — that's #13, not this

    ✅ This is about finding the single best pair of walls

  2. Moving the taller wall

    ❌ Moving the taller wall can only decrease or maintain area — never improve

    ✅ Always move the shorter wall — it's the bottleneck

  3. Using height[left] * (right - left) instead of min(h[L], h[R]) * width

    ❌ Water overflows over the shorter wall — must use min

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify):

  • ✅ "Maximize area/product between two elements"
  • ✅ Result depends on both distance AND value of two elements
  • ✅ Greedy: one factor decreases (width), other might increase (height)
  • ✅ "Choose two lines/walls/points to optimize something"
