#26 — Company Org Chart — Max Depth

🔖 Pattern: Tree DFS (Recursion on Trees)

🎯 Real-World Scenario START HERE ▼

HR ne poochha: "Company hierarchy kitni levels deep hai?"

         3 (CEO)
        / \
       9   20 (VPs)
      /  \
     15   7  (Directors)

Answer: 3 levels (CEO → VP → Director)

This is LeetCode #104 — Maximum Depth of Binary Tree. Your gateway to tree thinking.

🧠 1. Problem Understanding STEP 1 ▼

Binary Tree basics:

  • Each node has at most 2 children: left and right
  • Top = root. No children = leaf. Depth = nodes on longest root-to-leaf path.

Node structure:

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int val) { this.val = val; }
}

Why trees are perfect for recursion:

  • Every subtree IS a tree — perfect recursive structure
  • Depth of tree = 1 + max(depth of left subtree, depth of right subtree)
  • This is "trust the function" from Problem #20!

Connection: Recursion (#20) meets a new data structure. DFS uses the call stack implicitly (#15).

💪 2. Brute Force Thinking STEP 2 ▼

BFS (level-order): Process level by level with a queue, count levels. O(n) time, O(n) space.

Works, but recursive DFS is simpler and more elegant for this problem.

🤔 Depth = 1 + max(left depth, right depth). Base: null = 0. That's 3 lines of recursion.

⚡ 3. Optimization — The Universal Tree Template STEP 3 ▼ 
int solve(TreeNode node) {
    if (node == null) return BASE_VALUE;     // base case
    int leftResult  = solve(node.left);      // recurse left
    int rightResult = solve(node.right);     // recurse right
    return COMBINE(leftResult, rightResult); // combine
}

For max depth: BASE = 0, COMBINE = 1 + max(left, right).

This template solves 80% of tree problems. Change BASE and COMBINE:

Problem BASE COMBINE
Max Depth (this)01 + max(L, R)
Count Nodes01 + L + R
Sum of Nodes0node.val + L + R
Is Balanced (#27)0Check |L-R| ≤ 1, return height

Time:O(n)

Visit every node once. Space = recursion depth = height. Balanced: O(log n). Skewed: O(n).

☕ 8. Java Implementation STEP 8 ▼ 
public class MaxDepth {

    public static int maxDepth(TreeNode root) {
        if (root == null) return 0;
        int left  = maxDepth(root.left);
        int right = maxDepth(root.right);
        return 1 + Math.max(left, right);
    }

    // One-liner:
    // return root == null ? 0 : 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}

💡 3 lines of logic. The simplest tree problem — but the template it teaches is used everywhere.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting null check → NullPointerException on root.left
  2. Trying to trace every recursive call → Trust the function. Verify base + combine. Done.
🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use: ✅ Max/min depth, count nodes, sum nodes, any "answer = f(left answer, right answer)" tree problem.

Don't use: ❌ Level-order traversal → BFS. ❌ Very deep trees (>10K) → iterative DFS with explicit stack.
