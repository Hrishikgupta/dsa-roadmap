#29 — Serialize & Deserialize the Org Chart

🔖 Pattern: Tree BFS + Encoding

🎯 Real-World Scenario START HERE ▼

Tumhe company ka org chart (tree) ek file mein save karna hai (serialize) aur baad mein file se wapas tree banana hai (deserialize).

Tree:        Serialized string:
     1
    / \       "1,2,3,null,null,4,5"
   2   3
      / \     Deserialized back:
     4   5         same tree ✓

Yeh real-world mein hota hai: database mein tree store karna, network pe tree bhejna, cache mein tree save karna.

This is LeetCode #297 — Serialize and Deserialize Binary Tree. Hard-rated but the BFS approach is very intuitive.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Serialize: Convert a binary tree into a string
  • Deserialize: Convert the string back into the exact same tree
  • The encoding format is your choice — just ensure round-trip works: deserialize(serialize(tree)) == tree

Two main approaches:

Approach Traversal Null handling Complexity
DFS (Pre-order)Root, Left, RightMark nulls with "null"O(n) / O(n)
BFS (Level-order)Level by levelMark nulls with "null"O(n) / O(n)

We'll use BFS — it produces the same format as LeetCode's tree representation, making it intuitive.

Connection to previous problems:

  • #18: Queue for FIFO processing → BFS uses Queue to process level by level
  • #26-28: DFS on trees → now we learn BFS on trees
💪 2. Brute Force Thinking STEP 2 ▼

Why can't we just store in-order traversal?

  • In-order of [1,2,3,4,5] = "2,1,4,3,5"
  • But multiple trees can have the same in-order traversal!
  • We need to encode the STRUCTURE (which nodes are null) — not just values

Solution: Include null markers. BFS naturally visits level by level, including null children.

⚡ 3. Optimization Thinking STEP 3 ▼

BFS Serialize:

  1. Start with root in queue
  2. For each node in queue:
    • If not null → add value to string, enqueue left and right children
    • If null → add "null" to string

BFS Deserialize:

  1. Split string by comma → array of tokens
  2. First token = root. Add root to queue.
  3. For each node in queue, next two tokens are its left and right children
  4. If token is "null" → child is null. Else → create node, enqueue it.

💡 Pattern: BFS level-order with null markers — serialize writes level by level, deserialize reads level by level

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Key insight for deserialize: In BFS order, children come in pairs. For each parent dequeued, the next two tokens in the string are its left and right children.

This works because BFS processes parents in the same order they were serialized — FIFO guarantees alignment.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why BFS alignment works:

  • Serialize: parents are written in BFS order. For each parent, left child then right child are written.
  • Deserialize: parents are read in BFS order (queue). For each parent, next two tokens = its children.
  • Both use FIFO → the order matches perfectly.

BFS on trees — the template:

Queue<TreeNode> queue = new LinkedList<>();
queue.add(root);
while (!queue.isEmpty()) {
    TreeNode node = queue.poll();
    // process node
    if (node.left != null) queue.add(node.left);
    if (node.right != null) queue.add(node.right);
}

This BFS template is used in #29 (serialize), #30 (right view), #31-32 (graph BFS). Master it here.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(n)
Space:O(n)

Both serialize and deserialize visit each node once. Queue holds at most one level of nodes (max width).

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class Codec {

    public String serialize(TreeNode root) {
        if (root == null) return "";
        StringBuilder sb = new StringBuilder();
        Queue<TreeNode> queue = new LinkedList<>();
        queue.add(root);

        while (!queue.isEmpty()) {
            TreeNode node = queue.poll();
            if (node == null) {
                sb.append("null,");
            } else {
                sb.append(node.val).append(",");
                queue.add(node.left);
                queue.add(node.right);
            }
        }
        return sb.toString();
    }

    public TreeNode deserialize(String data) {
        if (data.isEmpty()) return null;
        String[] tokens = data.split(",");
        TreeNode root = new TreeNode(Integer.parseInt(tokens[0]));
        Queue<TreeNode> queue = new LinkedList<>();
        queue.add(root);
        int i = 1;

        while (!queue.isEmpty()) {
            TreeNode node = queue.poll();

            if (!tokens[i].equals("null")) {
                node.left = new TreeNode(Integer.parseInt(tokens[i]));
                queue.add(node.left);
            }
            i++;

            if (!tokens[i].equals("null")) {
                node.right = new TreeNode(Integer.parseInt(tokens[i]));
                queue.add(node.right);
            }
            i++;
        }
        return root;
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Not enqueueing null children during serialize

    ❌ Skipping nulls → lose structure info → can't deserialize

    ✅ Enqueue null children too — write "null" for them

  2. Token index misalignment during deserialize

    ✅ Always consume exactly 2 tokens per dequeued parent (left + right)

  3. Trailing comma or empty token after split

    ✅ Be careful with split(",") — trailing comma may create empty last token

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Serialize/Deserialize a tree" — save to file/network/database
  • ✅ "Clone a tree" — serialize + deserialize = deep copy
  • ✅ "Compare two trees across systems" — serialize both, compare strings
  • ✅ BFS template reused in: right view (#30), level-order traversal, zigzag traversal
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ BST — can serialize with just pre-order (no null markers needed, structure is implied by BST property)
  • ❌ N-ary tree — needs different encoding (child count per node)
💡 "Serialize = BFS writing values + null markers. Deserialize = BFS reading tokens in pairs (left, right) per parent. Queue keeps parent-child alignment."
