#37 — N-Queens — Place Queens Safely

🔖 Pattern: Backtracking (Placement + Pruning)

🎯 Real-World Scenario START HERE ▼

Place N queens on an N×N chessboard so that no two queens attack each other:

4-Queens solution:          Queens attack:
. Q . .                     ↑ ↗
. . . Q                   ← Q →  (row, col, diagonals)
Q . . .                     ↙ ↓
. . Q .

No two queens share the same row, column, or diagonal.

This is LeetCode #51 — N-Queens. The most famous backtracking problem in CS history.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Place N queens on N×N board — one queen per row (since each row must have exactly one)
  • No two queens in same column, same diagonal, or same anti-diagonal
  • Return all valid configurations

Key simplification: Since each row gets exactly one queen, we just need to decide WHICH COLUMN for each row. This reduces the problem from N² choices to N choices per row.

The diagonal trick:

  • All cells on the same diagonal (↘) have the same row - col value
  • All cells on the same anti-diagonal (↙) have the same row + col value
  • Track used columns, diagonals, and anti-diagonals in HashSets → O(1) conflict check

Connection to #36: Same backtracking template. Sudoku: try digits 1-9, check row/col/box. N-Queens: try columns 0 to N-1, check col/diagonal/anti-diagonal.

💪 2. Brute Force Thinking STEP 2 ▼

Brute force: Try all N^N column assignments (N choices for each of N rows), check validity.

For N=8: 8⁸ = 16 million. With backtracking + pruning: only ~15,000 nodes explored.

🤔 Place queen row by row. For each row, try each column. If column/diagonal/anti-diagonal is already taken → skip (prune). If valid → place and move to next row.

⚡ 3. Optimization Thinking STEP 3 ▼

Algorithm:

void solve(row, cols, diags, antiDiags, board, results):
    if row == N → add board to results; return

    for col = 0 to N-1:
        if col in cols || (row-col) in diags || (row+col) in antiDiags:
            continue  // PRUNE — conflict!

        // PLACE
        cols.add(col); diags.add(row-col); antiDiags.add(row+col)
        board[row][col] = 'Q'

        solve(row+1, ...)  // RECURSE to next row

        // UNDO
        cols.remove(col); diags.remove(row-col); antiDiags.remove(row+col)
        board[row][col] = '.'

💡 Three HashSets for O(1) conflict detection: cols, diags (row-col), antiDiags (row+col).

🎯 4. Final Approach STEP 4 ▼

Row-by-row placement:

  1. Process one row at a time (row 0, then 1, then 2...)
  2. For each row, try every column
  3. Check: is this column, diagonal, anti-diagonal free?
  4. If yes → place queen, add to sets, recurse to next row
  5. After recursion → undo (remove from sets, clear cell)
  6. When row == N → all queens placed, save the board
🔍 5. Step-by-Step Dry Run STEP 5 ▼

N = 4:

Row 0: try col 0
  Q . . .   cols={0}, diags={0}, anti={0}
  Row 1: try col 0 → conflict (col). col 1 → conflict (anti-diag 0+1=1, diag 1-1=0). col 2:
    Q . . .
    . . Q .   cols={0,2}, diags={0,-1}, anti={0,3}
    Row 2: col 0→conflict(col). col 1→conflict(anti 2+1=3). col 2→conflict(col). col 3→conflict(diag 2-3=-1).
    ALL BLOCKED → backtrack! Undo row 1 col 2.
  Row 1: try col 3:
    Q . . .
    . . . Q   cols={0,3}, diags={0,-2}, anti={0,4}
    Row 2: col 0→conflict(col). col 1:
      Q . . .
      . . . Q
      . Q . .   cols={0,3,1}, diags={0,-2,1}, anti={0,4,3}
      Row 3: col 0→conflict(col). col 1→conflict(col). col 2:
        diag 3-2=1 → conflict! col 3→conflict(col).
        ALL BLOCKED → backtrack!
  ... eventually finds:
  . Q . .
  . . . Q
  Q . . .
  . . Q .   ✓ SOLUTION FOUND!

For N=4, there are exactly 2 solutions. The algorithm finds both by continuing after the first.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why row-col and row+col identify diagonals:

4×4 board — row-col values (↘ diagonal):
 0  -1  -2  -3
 1   0  -1  -2
 2   1   0  -1
 3   2   1   0

4×4 board — row+col values (↙ anti-diagonal):
 0   1   2   3
 1   2   3   4
 2   3   4   5
 3   4   5   6

Same value = same diagonal. Store in HashSet → O(1) check.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(N!)
Space:O(N)

Row 0: N choices. Row 1: at most N-1 (one col taken). Row 2: at most N-2... ≈ N!. Space: 3 sets + recursion depth = O(N).

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class NQueens {

    public List<List<String>> solveNQueens(int n) {
        List<List<String>> results = new ArrayList<>();
        char[][] board = new char[n][n];
        for (char[] row : board) Arrays.fill(row, '.');
        solve(0, n, board, new HashSet<>(), new HashSet<>(), new HashSet<>(), results);
        return results;
    }

    private void solve(int row, int n, char[][] board,
                        Set<Integer> cols, Set<Integer> diags,
                        Set<Integer> antiDiags, List<List<String>> results) {
        if (row == n) {
            List<String> snapshot = new ArrayList<>();
            for (char[] r : board) snapshot.add(new String(r));
            results.add(snapshot);
            return;
        }

        for (int col = 0; col < n; col++) {
            int diag = row - col, anti = row + col;
            if (cols.contains(col) || diags.contains(diag)
                || antiDiags.contains(anti)) continue;

            board[row][col] = 'Q';
            cols.add(col); diags.add(diag); antiDiags.add(anti);

            solve(row + 1, n, board, cols, diags, antiDiags, results);

            board[row][col] = '.';
            cols.remove(col); diags.remove(diag); antiDiags.remove(anti);
        }
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Checking only row and column, forgetting diagonals

    ✅ Must check all three: column, diagonal (row-col), anti-diagonal (row+col)

  2. Not undoing all three sets during backtrack

    ❌ Removing from cols but forgetting diags/antiDiags → false conflicts in later branches

  3. Checking rows explicitly

    Not needed — we place exactly one queen per row by design (row-by-row recursion).

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Place N items with no-conflict constraints"
  • ✅ "Find all valid configurations" on a board/grid
  • ✅ "Graph coloring" — assign colors so no adjacent nodes share a color
  • ✅ Any constraint satisfaction problem (CSP)
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Count solutions" for large N — backtracking is too slow, need math/DP
  • ❌ No constraints to prune — degenerates to brute force
