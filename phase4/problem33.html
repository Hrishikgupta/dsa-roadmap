#33 — Course Prerequisite Order

🔖 Pattern: Topological Sort (Kahn's BFS)

🎯 Real-World Scenario START HERE ▼

Tum college mein course registration kar rahe ho. Kuch courses ke prerequisites hain:

4 courses (0,1,2,3), prerequisites:
To take 1, must complete 0 first:  0 → 1
To take 2, must complete 0 first:  0 → 2
To take 3, must complete 1 first:  1 → 3
To take 3, must complete 2 first:  2 → 3

Graph:
  0 → 1 → 3
  ↓       ↑
  2 ------┘

Valid order: [0, 1, 2, 3] or [0, 2, 1, 3]

Question: "Find a valid order to take all courses, respecting prerequisites."

This is LeetCode #210 — Course Schedule II. Also LC #207 (just check if possible). Classic Topological Sort.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given numCourses and prerequisites[][] (pairs [a,b] meaning "b before a")
  • Return a valid ordering of all courses, or empty array if impossible (cycle exists)

What is Topological Sort?

  • A linear ordering of nodes in a Directed Acyclic Graph (DAG)
  • For every edge u → v, u appears before v in the ordering
  • Only possible if graph has NO cycles (if cycle exists → impossible)

Key concept — In-degree:

  • In-degree of a node = number of edges pointing INTO it = number of prerequisites
  • A course with in-degree 0 = no prerequisites → can be taken immediately
  • After taking a course, reduce in-degree of its dependents
  • When a dependent's in-degree becomes 0 → it can now be taken

Connection to previous:

  • #32: BFS on grid (level by level). Here: BFS on directed graph (layer by layer of "ready" courses)
  • Same Queue-based BFS template, but nodes are courses and edges are prerequisites
💪 2. Brute Force Thinking STEP 2 ▼

Naive: Try all permutations of courses, check if each respects prerequisites. O(n!) — absurd.

🤔 Better: think like a student. "Which courses can I take RIGHT NOW?" → ones with no prerequisites (in-degree 0). Take them, then check what new courses become available. Repeat.

⚡ 3. Optimization Thinking STEP 3 ▼

Kahn's Algorithm (BFS-based Topological Sort):

  1. Compute in-degree for every node
  2. Add all nodes with in-degree 0 to queue (no prerequisites)
  3. While queue not empty:
    • Dequeue node → add to result order
    • For each neighbor of this node: decrement their in-degree
    • If any neighbor's in-degree becomes 0 → add to queue
  4. If result has all n nodes → valid order. If fewer → cycle exists (impossible).
// Kahn's Algorithm — BFS Topological Sort
Queue<Integer> queue = new LinkedList<>();
for (int i = 0; i < n; i++)
    if (inDegree[i] == 0) queue.add(i);

while (!queue.isEmpty()) {
    int course = queue.poll();
    result.add(course);
    for (int next : graph[course]) {
        inDegree[next]--;
        if (inDegree[next] == 0) queue.add(next);
    }
}

💡 Pattern: Kahn's Algorithm — BFS with in-degree tracking. Process nodes with 0 in-degree, reduce neighbors, repeat.

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Setup:

  1. Build adjacency list: graph[u] = list of courses that depend on u
  2. Compute inDegree[v] for each course

BFS:

  1. Enqueue all courses with inDegree == 0
  2. Process queue: dequeue → add to result → decrement neighbors' inDegree → enqueue if 0
  3. If result.size() == numCourses → return result. Else → cycle exists, return [].

⚙️ 6. Internal Mechanics STEP 6 ▼

Why Kahn's detects cycles:

  • In a cycle: A→B→C→A. Each has in-degree ≥ 1.
  • None of them ever reaches in-degree 0 → none gets enqueued → none gets processed
  • Result has fewer than n nodes → cycle detected!

Kahn's BFS vs DFS Topological Sort:

ApproachRemove nodes with 0 in-degreePost-order DFS, reverse result

Real-world topological sort:

  • Build systems (Makefile, Gradle) — compile dependencies in order
  • Task scheduling — which tasks can run in parallel?
  • Package managers (npm, pip) — install dependencies first
  • Spreadsheet cell evaluation — compute cells in dependency order

Time:O(V + E)

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class CourseOrder {

    public static int[] findOrder(int numCourses, int[][] prereqs) {
        List<List<Integer>> graph = new ArrayList<>();
        int[] inDegree = new int[numCourses];

        for (int i = 0; i < numCourses; i++) graph.add(new ArrayList<>());

        for (int[] p : prereqs) {
            graph.get(p[1]).add(p[0]); // p[1] → p[0]
            inDegree[p[0]]++;
        }

        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (inDegree[i] == 0) queue.add(i);
        }

        int[] result = new int[numCourses];
        int idx = 0;

        while (!queue.isEmpty()) {
            int course = queue.poll();
            result[idx++] = course;
            for (int next : graph.get(course)) {
                if (--inDegree[next] == 0) queue.add(next);
            }
        }

        return idx == numCourses ? result : new int[0];
    }
}

💡 The idx == numCourses check at the end = cycle detection. If we couldn't process all courses → cycle exists → return empty.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Wrong edge direction

    ❌ prereq [1,0] means "to take 1, need 0 first" → edge is 0→1, NOT 1→0

    ✅ graph.get(p[1]).add(p[0]) and inDegree[p[0]]++

  2. Forgetting cycle detection

    ❌ Returning result without checking if all courses were processed

    ✅ return idx == numCourses ? result : new int[0]

  3. Not initializing adjacency list

    ✅ Create empty list for EVERY node, even those with no outgoing edges

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Course schedule" / "Task ordering" / "Build order"
  • ✅ "Is it possible to finish all tasks?" (cycle detection)
  • ✅ "Dependency resolution" — any DAG ordering problem
  • ✅ "Alien dictionary" — derive letter order from sorted words
  • ✅ Any problem with "must do X before Y" constraints
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Undirected graph — topological sort is only for directed graphs
  • ❌ Graph has cycles — no valid topological order exists (but Kahn's detects this)
  • ❌ "Shortest path" — use BFS (#32) or Dijkstra (#35), not topo sort
💡 "Topological Sort = BFS with in-degree. Start with 0-inDegree nodes, process them, reduce neighbors' inDegree, enqueue new 0s. If not all processed → cycle."
