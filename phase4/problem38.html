#38 — Word Search in a Grid

🔖 Pattern: Backtracking + DFS on Grid

🎯 Real-World Scenario START HERE ▼

Tum ek word puzzle game bana rahe ho (like Boggle). Grid mein letters hain:

A B C E
S F C S
A D E E

Find "ABCCED":
A → B → C → C → E → D  ✓ (path exists!)

Find "SEE":
S → E → E  ✓

Find "ABCB":
A → B → C → B?  ✗ (can't revisit B!)

Rules: Start anywhere, move to adjacent cells (up/down/left/right), can't revisit same cell in one path.

This is LeetCode #79 — Word Search. Combines grid DFS (#31) with backtracking (#36).

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a 2D grid of characters and a word, determine if the word exists in the grid
  • Word is formed by sequentially adjacent cells (horizontal/vertical)
  • Same cell cannot be used more than once in a single path

This combines two patterns:

From Pattern Used Here
#31Grid DFS (4-directional)Explore adjacent cells
#36Backtracking (undo)Mark visited → explore → UNMARK (allow reuse in other paths)

Key difference from #31 (Islands):

  • #31: Mark visited permanently (sink the island). Never undo.
  • #38: Mark visited temporarily. UNDO after exploring (backtrack). Same cell can be used in different paths.
💪 2. Brute Force Thinking STEP 2 ▼

Approach: For each cell that matches the first character, start a DFS. At each step, check if current cell matches the next character. If yes → continue. If no → backtrack.

🤔 This IS the optimal approach — there's no way to avoid exploring paths. Backtracking with pruning (wrong character = stop immediately) makes it efficient enough.

⚡ 3. Optimization Thinking STEP 3 ▼

Algorithm:

boolean dfs(board, row, col, word, index):
    if index == word.length → return true  // all chars matched!

    if out of bounds || board[row][col] != word[index]:
        return false  // PRUNE: wrong char or out of grid

    char temp = board[row][col]
    board[row][col] = '#'  // MARK visited (temporary)

    boolean found = dfs(row-1,col,...) || dfs(row+1,col,...)
                 || dfs(row,col-1,...) || dfs(row,col+1,...)

    board[row][col] = temp  // UNMARK (backtrack!)
    return found

The "temporary marking" trick:

  • Replace cell with '#' (or any non-letter) to mark as "in current path"
  • After exploring all 4 directions → restore original character
  • This prevents revisiting in the CURRENT path but allows reuse in OTHER paths
  • No separate visited array needed — modify grid in-place, then undo

💡 Pattern: Grid DFS + Backtracking — mark temporarily, explore 4 directions, unmark. Character matching = pruning.

🎯 4. Final Approach STEP 4 ▼

Two-level structure:

  1. Outer: Scan grid for cells matching word[0] → start DFS from each
  2. Inner (DFS): At each cell, check character match → mark → recurse 4 directions → unmark

Short-circuit with ||: if any direction returns true, stop immediately (word found).

📊 7. Complexity Analysis STEP 7 ▼
Time:O(m·n·3^L)
Space:O(L)

m×n starting cells. At each step, 3 directions (can't go back where we came from). L = word length. Space = recursion depth = L.

☕ 8. Java Implementation STEP 8 ▼ 
public class WordSearch {

    public boolean exist(char[][] board, String word) {
        int m = board.length, n = board[0].length;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if (dfs(board, i, j, word, 0, m, n)) return true;
        return false;
    }

    private boolean dfs(char[][] board, int r, int c,
                          String word, int idx, int m, int n) {
        if (idx == word.length()) return true;
        if (r < 0 || r >= m || c < 0 || c >= n
            || board[r][c] != word.charAt(idx)) return false;

        char temp = board[r][c];
        board[r][c] = '#'; // mark visited

        boolean found = dfs(board, r-1, c, word, idx+1, m, n)
                     || dfs(board, r+1, c, word, idx+1, m, n)
                     || dfs(board, r, c-1, word, idx+1, m, n)
                     || dfs(board, r, c+1, word, idx+1, m, n);

        board[r][c] = temp; // BACKTRACK — restore
        return found;
    }
}

💡 The || short-circuits: if any direction finds the word, we stop immediately. The backtrack (restore) still happens because Java evaluates the assignment before returning.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Not restoring the cell after DFS (permanent marking)

    ❌ Cell stays '#' → other starting points can't use it → misses valid paths

    ✅ Always restore: board[r][c] = temp after exploring

  2. Checking index AFTER bounds/character check

    ❌ if (board[r][c] != word[idx]) before if (idx == word.length()) → index out of bounds

    ✅ Check idx == word.length() FIRST (base case before validation)

  3. Using a separate visited array instead of in-place marking

    Works but uses O(m×n) extra space. In-place '#' marking is cleaner and O(1) extra.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Word search in grid" / "Path exists in grid matching pattern"
  • ✅ "Find path with specific sequence" in a matrix
  • ✅ Any grid problem where you need to explore paths and undo visited state
  • ✅ "Word Search II" (LC #212) — same but with Trie optimization for multiple words
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Shortest path" in grid — use BFS (#32), not DFS backtracking
  • ❌ "Count connected components" — use permanent marking (#31), not temporary
  • ❌ Very long words on large grids — exponential worst case, may TLE
💡 "Word Search = Grid DFS + Backtracking. Mark cell temporarily ('#'), explore 4 directions, restore after. Character mismatch = instant prune."
