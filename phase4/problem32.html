#32 — Shortest Path in a Maze

🔖 Pattern: Graph BFS (Shortest Path Unweighted)

🎯 Real-World Scenario START HERE ▼

Tum ek maze game bana rahe ho. Player ko top-left se bottom-right tak pahunchna hai:

0 = open path, 1 = wall

S 0 0 0
1 1 0 1
0 0 0 0
0 1 1 E

S = Start (0,0)
E = End (3,3)

Shortest path: S→(0,1)→(0,2)→(1,2)→(2,2)→(2,3)→(3,3) = 6 steps

Question: "Minimum kitne steps mein player exit tak pahunch sakta hai?"

This is the classic BFS shortest path on grid. Foundation for LC #1091, #994, #542, and many more.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a grid with 0 (open) and 1 (wall), find shortest path from (0,0) to (m-1,n-1)
  • Can move up/down/left/right (4 directions)
  • Return number of steps, or -1 if no path exists

Why BFS, not DFS?

Aspect DFS BFS
FindsA path (any path)THE shortest path
ExploresDeep firstLevel by level (distance 1, then 2, then 3...)
GuaranteeNo shortest guaranteeFirst time reaching target = shortest
Use forExistence, counting, backtrackingShortest path (unweighted)

💡 Golden rule: BFS = shortest path in unweighted graphs. Each BFS level = one more step from start.

Connection to previous:

  • #31: Grid DFS for connected components (explore all)
  • #32: Grid BFS for shortest path (explore level by level)
  • #29-30: Tree BFS template → same Queue-based approach, now on a grid
💪 2. Brute Force Thinking STEP 2 ▼

DFS approach: Try all paths, track minimum length. Works but explores many unnecessary long paths.

BFS approach: Explore all cells at distance 1, then distance 2, then 3... First time you reach the target = shortest. No need to explore further.

🤔 BFS is like a ripple in water — it expands outward uniformly. The first ripple to reach the target = shortest distance.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Ripple/Wave Analogy:

  • Drop a stone at the start cell. Ripple expands outward.
  • After 1 second: all cells at distance 1 are reached
  • After 2 seconds: all cells at distance 2 are reached
  • The moment the ripple hits the target → that's the shortest distance

BFS Algorithm:

Queue<int[]> queue = new LinkedList<>();
queue.add({startRow, startCol});
visited[startRow][startCol] = true;
int steps = 0;

while (!queue.isEmpty()) {
    int size = queue.size();  // nodes at current distance
    for (int i = 0; i < size; i++) {
        int[] cell = queue.poll();
        if (cell == target) return steps;  // FOUND!
        for each neighbor:
            if valid && !visited:
                visited = true;
                queue.add(neighbor);
    }
    steps++;  // next distance level
}

💡 Pattern: BFS with level-size loop — each level = one more step. First time reaching target = shortest path.

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Algorithm:

  1. If start or end is a wall → return -1
  2. Add start to queue, mark visited
  3. BFS with level-size loop:
    • For each cell in current level: check if target → return steps
    • Add all valid unvisited neighbors to queue, mark visited
    • Increment steps after processing level
  4. Queue empty without reaching target → return -1

⚠️ Critical: mark visited WHEN ADDING to queue, not when processing. Otherwise the same cell gets added multiple times from different neighbors.

🔍 5. Step-by-Step Dry Run STEP 5 ▼

Grid:

S 0 0 0
1 1 0 1
0 0 0 0
0 1 1 E
Steps Queue (cells at this distance) New neighbors added
0(0,0)→ (0,1)
1(0,1)→ (0,2)
2(0,2)→ (0,3), (1,2)
3(0,3), (1,2)→ (2,2)
4(2,2)→ (2,1), (2,3)
5(2,1), (2,3)→ (3,0) from (2,1), (3,3) from (2,3)
6(3,0), (3,3)(3,3) = TARGET → return 6

Result: 6 steps ✓

BFS explored level by level. The moment (3,3) was dequeued at step 6, we knew that's the shortest.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why BFS guarantees shortest path:

  • BFS explores ALL cells at distance d before ANY cell at distance d+1
  • So the first time we reach the target, we've taken the minimum possible steps
  • No shorter path can exist — we would have found it at an earlier level

Mark visited WHEN ADDING vs WHEN PROCESSING:

// WRONG: mark when processing
cell = queue.poll();
visited[cell] = true;  // too late! cell might be in queue multiple times

// CORRECT: mark when adding
visited[nr][nc] = true;  // mark immediately
queue.add({nr, nc});     // then add — no duplicates possible

BFS shortest path — the universal template:

// Works for any unweighted graph (grid, adjacency list, etc.)
1. Add start to queue, mark visited
2. while queue not empty:
     levelSize = queue.size()
     for i = 0 to levelSize-1:
       node = queue.poll()
       if node == target → return steps
       for each neighbor:
         if valid && !visited → mark, add to queue
     steps++
3. return -1  // unreachable
📊 7. Complexity Analysis STEP 7 ▼
Time:O(m × n)
Space:O(m × n)

Each cell visited at most once. Queue + visited array = O(m×n).

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class ShortestPathMaze {

    public static int shortestPath(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        if (grid[0][0] == 1 || grid[m-1][n-1] == 1) return -1;

        int[] dr = {-1,1,0,0}, dc = {0,0,-1,1};
        boolean[][] visited = new boolean[m][n];
        Queue<int[]> queue = new LinkedList<>();

        queue.add(new int[]{0, 0});
        visited[0][0] = true;
        int steps = 0;

        while (!queue.isEmpty()) {
            int size = queue.size();
            for (int i = 0; i < size; i++) {
                int[] cell = queue.poll();
                if (cell[0] == m-1 && cell[1] == n-1) return steps;

                for (int d = 0; d < 4; d++) {
                    int nr = cell[0]+dr[d], nc = cell[1]+dc[d];
                    if (nr>=0 && nr<m && nc>=0 && nc<n
                        && !visited[nr][nc] && grid[nr][nc]==0) {
                        visited[nr][nc] = true;
                        queue.add(new int[]{nr, nc});
                    }
                }
            }
            steps++;
        }
        return -1;
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Marking visited when PROCESSING instead of when ADDING

    ❌ Same cell added to queue multiple times → TLE and wrong answer

    ✅ Mark visited immediately when adding to queue

  2. Using DFS for shortest path

    ❌ DFS finds A path, not THE shortest. Would need to explore ALL paths and take minimum.

    ✅ BFS guarantees shortest in unweighted graphs

  3. Forgetting to check start/end validity

    ✅ Check if start or end is a wall before starting BFS

  4. Off-by-one in step counting

    Steps = 0 at start. Increment AFTER processing each level. Check target WHEN processing, not when adding.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Shortest path" in unweighted graph/grid
  • ✅ "Minimum steps/moves to reach target"
  • ✅ "Rotting oranges" / "Walls and gates" (multi-source BFS)
  • ✅ "Word ladder" (shortest transformation sequence)
  • ✅ Any problem where each step has equal cost

When edges have different weights → Dijkstra (#35), not BFS.

🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Weighted edges (different costs) → use Dijkstra or Bellman-Ford
  • ❌ "Find ALL paths" → use DFS + backtracking
  • ❌ "Longest path" → NP-hard in general, BFS doesn't help
💡 "Shortest path in unweighted graph = BFS. Level-by-level expansion. First time reaching target = shortest. Mark visited when ADDING, not when processing."
