#27 — Is This a Balanced Team Structure?

🔖 Pattern: Tree DFS (Height + Validation)

This is LeetCode #110 — Balanced Binary Tree.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • A binary tree is height-balanced if for EVERY node, the heights of left and right subtrees differ by at most 1
  • Not just the root — EVERY node must satisfy this condition

Input → Output:

  • Input: TreeNode root
  • Output: boolean

Connection to #26:

  • #26: compute height → 1 + max(L, R)
  • #27: compute height + validate → same computation, but also check |L - R| ≤ 1 at every node
  • If any node fails → return "unbalanced" signal immediately
💪 2. Brute Force Thinking STEP 2 ▼

Naive approach — O(n²):

boolean isBalanced(TreeNode root) {
    if (root == null) return true;
    int leftH  = maxDepth(root.left);   // O(n) each call
    int rightH = maxDepth(root.right);  // O(n) each call
    return Math.abs(leftH - rightH) <= 1
        && isBalanced(root.left)          // recurse
        && isBalanced(root.right);         // recurse
}

For each node, we call maxDepth which traverses the subtree → O(n) per node × n nodes = O(n²).

🤔 We're computing height repeatedly. Can we compute height AND check balance in a single pass?

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — Return height, but use -1 as "unbalanced" signal:

  • Normal height function returns 0, 1, 2, 3... (always ≥ 0)
  • If we detect imbalance at any node → return -1 (special sentinel)
  • If any child returns -1 → propagate -1 upward (don't bother computing further)
  • At the end: if root's height is -1 → unbalanced. Otherwise → balanced.
int checkHeight(TreeNode node):
    if node == null → return 0

    int leftH = checkHeight(node.left)
    if leftH == -1 → return -1          // left subtree unbalanced — abort

    int rightH = checkHeight(node.right)
    if rightH == -1 → return -1         // right subtree unbalanced — abort

    if |leftH - rightH| > 1 → return -1 // THIS node unbalanced — abort

    return 1 + max(leftH, rightH)        // balanced — return real height

💡 Pattern: DFS with sentinel return value — compute height normally, return -1 to signal failure. One pass, O(n).

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Tree DFS — Height computation with early termination via sentinel (-1)

Algorithm:

  1. Define checkHeight(node) that returns height or -1
  2. Base: null → 0
  3. Get left height. If -1 → return -1 (early abort)
  4. Get right height. If -1 → return -1 (early abort)
  5. If |leftH - rightH| > 1 → return -1 (this node fails)
  6. Else → return 1 + max(leftH, rightH)

isBalanced(root) = checkHeight(root) != -1

⚙️ 6. Internal Mechanics STEP 6 ▼

The sentinel pattern — returning extra info through the return value:

  • Normal return: height (0, 1, 2, ...) → everything is fine
  • Sentinel return: -1 → something went wrong (unbalanced)
  • This avoids needing a separate boolean + height (two return values)
  • Same technique used in many tree problems: return -1 for invalid, real value for valid

O(n²) vs O(n) comparison:

Approach Time Why
Naive (height per node)O(n²)Recomputes height at every node
Sentinel (-1)O(n)Computes height once, validates during same pass
📊 7. Complexity Analysis STEP 7 ▼
Time:O(n)
Space:O(h)

Each node visited once. Space = recursion depth = tree height.

☕ 8. Java Implementation STEP 8 ▼ 
public class BalancedTree {

    public static boolean isBalanced(TreeNode root) {
        return checkHeight(root) != -1;
    }

    private static int checkHeight(TreeNode node) {
        if (node == null) return 0;

        int leftH = checkHeight(node.left);
        if (leftH == -1) return -1;

        int rightH = checkHeight(node.right);
        if (rightH == -1) return -1;

        if (Math.abs(leftH - rightH) > 1) return -1;

        return 1 + Math.max(leftH, rightH);
    }
}

💡 Same as #26's maxDepth, plus 3 lines of early-abort checks. The sentinel -1 does all the heavy lifting.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Only checking root balance

    ❌ Root might be balanced but a subtree isn't

    ✅ The sentinel -1 propagates from ANY unbalanced node upward

  2. Using the O(n²) approach in interviews

    ⚠️ Works but interviewer will ask "can you do it in one pass?" → sentinel approach

  3. Not early-aborting on -1

    ❌ Computing right height even when left returned -1 → wasted work

    ✅ Check if (leftH == -1) return -1 BEFORE computing right

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Is the tree balanced / symmetric / same?"
  • ✅ Any tree validation that needs height + condition at every node
  • ✅ "Validate BST" — same DFS + range validation pattern
  • ✅ Whenever you need to return a value AND a boolean from DFS → use sentinel
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Balance the tree" (restructure it) — that's AVL/Red-Black tree rotation, different problem
  • ❌ "Is the tree complete?" — needs BFS level-order check, not DFS height
💡 "DFS + sentinel: compute height normally, return -1 on failure. One pass, O(n). The -1 propagates upward like an alarm — if any node fails, the whole tree fails."
