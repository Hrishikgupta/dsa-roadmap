🔖 Pattern: Backtracking (Constraint-Based)

🧠 1. Problem Understanding — What is Backtracking? STEP 1 ▼

Backtracking = Recursion + constraint checking + undo on failure

Aspect Recursion (#21-23) Backtracking (#36-38)
GoalGenerate ALL possibilitiesFind ONE valid solution (or all valid)
ConstraintsNone — explore everythingCheck before going deeper (prune)
PruningNoYes — skip invalid branches early

💡 Backtracking = recursion with "if invalid, don't go deeper" (pruning). Much faster than brute force.

💪 2. Brute Force Thinking STEP 2 ▼

Try all 9 digits in every empty cell, check board at end. 9^(empty cells) ≈ 9⁵⁰ ≈ 10⁴⁷. Impossible.

🤔 But most choices are invalid early. CHECK constraints immediately after placing → prune entire branches. That's backtracking — fail fast, try next.

⚡ 3. The Universal Backtracking Template STEP 3 ▼ 
boolean backtrack(state):
    if goal reached → return true

    for each choice:
        if isValid(choice):     // 1. CONSTRAINT CHECK (prune)
            make choice             // 2. PLACE
            if backtrack(next) → return true  // 3. RECURSE
            undo choice             // 4. UNDO (backtrack)

    return false                // 5. DEAD END

This template solves Sudoku (#36), N-Queens (#37), Word Search (#38), and many more. Only choices and constraints change.

For Sudoku specifically:

  • Choices: digits '1' to '9' for each empty cell
  • Constraints: number not in same row, column, or 3×3 box
  • Goal: no empty cells left
  • 3×3 box index: boxRow = 3*(row/3), boxCol = 3*(col/3)

💡 Pattern: Backtracking = for each choice: check → place → recurse → undo if failed.

When no valid number works for a cell → return false → previous cell undoes its choice and tries next number. That's the "undo and retry" of backtracking.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why backtracking is fast enough:

  • Brute force: 9⁵⁰ ≈ 10⁴⁷
  • Backtracking with pruning: typically ~10,000 operations for standard Sudoku
  • Constraint checking eliminates most branches immediately

☕ 8. Java Implementation STEP 8 ▼ 
public class SudokuSolver {

    public void solveSudoku(char[][] board) {
        solve(board);
    }

    private boolean solve(char[][] board) {
        for (int r = 0; r < 9; r++) {
            for (int c = 0; c < 9; c++) {
                if (board[r][c] != '.') continue;

                for (char num = '1'; num <= '9'; num++) {
                    if (isValid(board, r, c, num)) {
                        board[r][c] = num;          // place
                        if (solve(board)) return true; // recurse
                        board[r][c] = '.';          // undo
                    }
                }
                return false; // no valid number → backtrack
            }
        }
        return true; // all cells filled → solved!
    }

    private boolean isValid(char[][] board, int r, int c, char num) {
        for (int i = 0; i < 9; i++) {
            if (board[r][i] == num) return false;  // row
            if (board[i][c] == num) return false;  // col
            int br = 3*(r/3) + i/3, bc = 3*(c/3) + i%3;
            if (board[br][bc] == num) return false; // box
        }
        return true;
    }
}

💡 The return false after the num loop is the backtracking trigger — "tried all 9, none worked, go back."

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting to undo

    ❌ No board[r][c] = '.' → board stays modified for wrong branches

    ✅ Always undo after failed recursion

  2. Wrong 3×3 box calculation

    ✅ Box start: 3*(r/3), 3*(c/3). Iterate with i/3 and i%3 offsets.

  3. Not returning false after trying all 9 numbers

    ✅ return false after the for-loop triggers backtracking to previous cell

🔑 11. When to Use / 12. When NOT STEP 11-12 ▼

Use: ✅ "Solve a puzzle" (Sudoku, crossword) ✅ "Find valid configuration" (N-Queens, graph coloring) ✅ "Generate all valid ___" (parentheses, IP addresses) ✅ Choices + constraints + explore/undo

Don't use: ❌ Optimization (min/max) → usually DP ❌ No constraints to prune → degenerates to brute force ❌ Overlapping subproblems → use DP with memoization

💡 "Backtracking = recursion + constraint check + undo. Try → check → recurse → undo if failed. The constraint check (pruning) is what makes it fast."
