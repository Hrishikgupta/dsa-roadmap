#34 — Detect a Cycle in Task Dependencies

🔖 Pattern: Cycle Detection (DFS 3-Color)

🎯 Real-World Scenario START HERE ▼

Tum ek CI/CD pipeline design kar rahe ho. Tasks ke dependencies hain:

No cycle (valid):        Cycle (deadlock!):
  Build → Test → Deploy     Build → Test
                               ↑       ↓
                              Lint ← Deploy
                           (circular dependency = deadlock)

Question: "Kya is dependency graph mein cycle hai? Agar hai toh pipeline deadlock ho jayega."

This is LeetCode #207 — Course Schedule (can you finish all courses?). Also applies to deadlock detection in OS.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a directed graph, determine if it contains a cycle
  • If cycle exists → tasks can't all be completed (deadlock)
  • If no cycle → valid DAG, topological order exists

Two approaches to detect cycles:

Method How From Problem
Kahn's BFSIf processed < n → cycle#33 (implicit)
DFS 3-ColorIf we visit a node currently being processed → cycleThis problem

The 3-Color System:

  • ⚪ WHITE (0) = unvisited — haven't touched this node yet
  • 🟡 GRAY (1) = in progress — currently exploring this node's subtree (on the DFS stack)
  • 🟢 BLACK (2) = done — fully explored, all descendants processed

Cycle = visiting a GRAY node from another GRAY node. It means we've found a back edge — a path that loops back to a node still being processed.

💪 2. Brute Force Thinking STEP 2 ▼

Why simple "visited" boolean isn't enough:

  • With just visited=true/false, you can't distinguish between:
  • "I'm currently exploring this node's path" (GRAY) — revisiting = CYCLE
  • "I already fully explored this node" (BLACK) — revisiting = just a cross edge, NOT a cycle
Example: A → B → D, A → C → D
When exploring A→C→D, D is already visited (from A→B→D path)
But this is NOT a cycle — D is BLACK (fully done)
With just boolean visited, we'd wrongly report a cycle

🤔 We need THREE states, not two. That's why it's called "3-color" DFS.

⚡ 3. Optimization Thinking STEP 3 ▼

DFS 3-Color Algorithm:

boolean hasCycle(node):
    if color[node] == GRAY → return true   // CYCLE! back edge found
    if color[node] == BLACK → return false  // already done, skip

    color[node] = GRAY  // start processing

    for neighbor in graph[node]:
        if hasCycle(neighbor) → return true

    color[node] = BLACK  // done processing
    return false

The lifecycle of a node:

  • ⚪ WHITE → 🟡 GRAY (start DFS on this node)
  • 🟡 GRAY → explore all neighbors recursively
  • 🟡 GRAY → 🟢 BLACK (all neighbors done, backtrack)
  • If during exploration we hit a 🟡 GRAY node → we've looped back → CYCLE!

💡 Pattern: DFS 3-Color — WHITE→GRAY→BLACK. Hitting GRAY from GRAY = cycle (back edge).

🎯 4. Final Approach STEP 4 ▼

Algorithm:

  1. Initialize all nodes as WHITE (0)
  2. For each WHITE node: start DFS
  3. DFS: mark GRAY → explore neighbors → if any returns cycle → propagate true
  4. After all neighbors done → mark BLACK → return false
  5. If any DFS returns true → graph has cycle

💡 Must start DFS from EVERY unvisited node — graph might be disconnected. A cycle could be in any component.

⚙️ 6. Internal Mechanics STEP 6 ▼

Edge types in DFS:

Edge type Target color Meaning Cycle?
Tree edgeWHITEUnvisited → explore itNo
Back edgeGRAYStill being processed → loop!YES!
Cross/Forward edgeBLACKAlready fully done → skipNo

Kahn's (#33) vs DFS 3-Color (#34):

Aspect Kahn's BFS DFS 3-Color
Detects cycleImplicitly (count < n)Explicitly (back edge)
Gives topo orderYes (naturally)Yes (reverse post-order)
Can find the cycleNo (just knows it exists)Yes (trace GRAY nodes)
Undirected graphsDoesn't workNeeds modification (parent tracking)

Time:O(V + E)
Space:O(V + E)

Each node visited once (WHITE→GRAY→BLACK). Each edge checked once. Graph storage = O(V+E).

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class CycleDetection {

    public static boolean canFinish(int n, int[][] prereqs) {
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) graph.add(new ArrayList<>());
        for (int[] p : prereqs) graph.get(p[1]).add(p[0]);

        int[] color = new int[n]; // 0=WHITE, 1=GRAY, 2=BLACK

        for (int i = 0; i < n; i++) {
            if (color[i] == 0 && hasCycle(graph, i, color)) {
                return false; // cycle found → can't finish
            }
        }
        return true; // no cycle → can finish all courses
    }

    private static boolean hasCycle(List<List<Integer>> graph,
                                      int node, int[] color) {
        if (color[node] == 1) return true;  // GRAY → cycle!
        if (color[node] == 2) return false; // BLACK → done, skip

        color[node] = 1; // mark GRAY

        for (int next : graph.get(node)) {
            if (hasCycle(graph, next, color)) return true;
        }

        color[node] = 2; // mark BLACK
        return false;
    }
}

💡 canFinish returns true if NO cycle (can finish all courses). The DFS helper returns true if cycle IS found.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Using boolean visited instead of 3 colors

    ❌ Can't distinguish GRAY (in-progress) from BLACK (done) → false positives

    ✅ Use int[n] with 0/1/2 for WHITE/GRAY/BLACK

  2. Not starting DFS from every unvisited node

    ❌ Cycle might be in a disconnected component you never visited

    ✅ Loop through all nodes: if (color[i] == 0) dfs(i)

  3. Forgetting to mark BLACK after processing

    ❌ Node stays GRAY forever → false cycle detection on cross edges

    ✅ color[node] = 2 after all neighbors are done

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Detect cycle in directed graph"
  • ✅ "Can all tasks be completed?" (Course Schedule LC #207)
  • ✅ "Deadlock detection" in operating systems
  • ✅ "Is this a valid DAG?"
  • ✅ When you need to FIND the cycle (trace GRAY nodes), not just detect it
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Undirected graph cycle detection — use Union-Find or simple DFS with parent tracking
  • ❌ Just need topological order (no cycle info) — Kahn's BFS (#33) is simpler
  • ❌ "Shortest cycle" — needs BFS from each node, not DFS coloring
💡 "Cycle in directed graph = DFS 3-color. WHITE→GRAY→BLACK. If you visit a GRAY node → back edge → CYCLE. If you visit BLACK → already done → safe."
