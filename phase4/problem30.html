#30 — Right Side View of a Building

🔖 Pattern: Tree BFS (Level Order)

🎯 Real-World Scenario START HERE ▼

Imagine tum ek building ke RIGHT side se khade ho aur andar dekh rahe ho — har floor pe sirf rightmost room dikhega:

Tree:           Right side view:
     1          →  1
    / \
   2   3        →  3
    \   \
     5   4      →  4

Answer: [1, 3, 4]

Har level ka last node = right side se visible node.

This is LeetCode #199 — Binary Tree Right Side View.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a binary tree, return the values of nodes visible from the right side
  • Visible = the last node at each level (rightmost)

Input → Output:

  • Input: TreeNode root
  • Output: List<Integer> — one value per level (rightmost)

Two approaches:

Approach How Complexity
BFS (level-order)Process level by level, take last node of each levelO(n) / O(w)
DFS (right-first)DFS visiting right child first, track first node at each depthO(n) / O(h)

BFS is more intuitive — it naturally processes levels. Same template as #29.

Connection to #29: #29 processed ALL nodes per level. #30 only keeps the LAST node per level.

💪 2. Brute Force Thinking STEP 2 ▼

Naive: Do a full level-order traversal, store all levels, then pick the last element of each level.

Works but stores all nodes. We can optimize: during BFS, just track the last node of each level — no need to store entire levels.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — BFS with level size tracking:

  • At the start of each level, queue.size() = number of nodes in this level
  • Process exactly that many nodes (inner loop)
  • The LAST node processed in the inner loop = rightmost of this level
while (!queue.isEmpty()) {
    int levelSize = queue.size();  // nodes in this level
    for (int i = 0; i < levelSize; i++) {
        TreeNode node = queue.poll();
        if (i == levelSize - 1) result.add(node.val); // last = rightmost
        if (node.left != null) queue.add(node.left);
        if (node.right != null) queue.add(node.right);
    }
}

💡 Pattern: BFS with level-size loop — process exactly one level per outer iteration, pick last node

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Algorithm:

  1. If root is null → return empty list
  2. Add root to queue
  3. While queue not empty:
    • levelSize = queue.size()
    • For i = 0 to levelSize-1: dequeue, enqueue children
    • If i == levelSize - 1 → add to result (rightmost)

This "level-size BFS" template solves many problems:

  • Right side view: last node per level
  • Left side view: first node per level
  • Level averages: sum/count per level
  • Zigzag traversal: alternate left-right per level
  • Max width: max level size

⚙️ 6. Internal Mechanics STEP 6 ▼

Why queue.size() at the start of each iteration works:

  • Before the inner loop, queue contains ONLY nodes from the current level
  • Inner loop dequeues exactly levelSize nodes (current level) and enqueues their children (next level)
  • After inner loop, queue contains ONLY next level's nodes
  • This cleanly separates levels without any extra data structure

DFS alternative (right-first DFS):

void dfs(TreeNode node, int depth, List<Integer> result) {
    if (node == null) return;
    if (depth == result.size()) result.add(node.val); // first node at this depth
    dfs(node.right, depth + 1, result); // right FIRST
    dfs(node.left, depth + 1, result);
}

Visit right child first → first node encountered at each depth = rightmost. O(h) space vs O(w) for BFS.

📊 7. Complexity Analysis STEP 7 ▼
Time:O(n)
Space:O(w)

Visit every node once. Queue holds at most one level = max width w. For balanced tree: w ≈ n/2.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class RightSideView {

    public static List<Integer> rightSideView(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;

        Queue<TreeNode> queue = new LinkedList<>();
        queue.add(root);

        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                if (i == levelSize - 1) result.add(node.val);
                if (node.left != null) queue.add(node.left);
                if (node.right != null) queue.add(node.right);
            }
        }
        return result;
    }
}

💡 Change i == levelSize - 1 to i == 0 → left side view. Change to collect all → level-order traversal. Same template.

🧪 9. Edge Cases STEP 9 ▼
CaseOutputNote
null[]Empty tree
Single node [1][1]Root is the only visible node
Left-skewed 1→2→3[1,2,3]Left nodes visible (no right siblings to block)
Right-skewed 1→2→3[1,2,3]Right nodes naturally visible
Left subtree deeperIncludes left-side deep nodesRight side view ≠ right subtree only!

⚠️ Common misconception: "right side view = right subtree." Wrong! If left subtree is deeper, its deep nodes ARE visible from the right.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Only traversing the right subtree

    ❌ Misses left-subtree nodes that are visible when right subtree is shorter

    ✅ BFS processes ALL nodes per level — naturally handles this

  2. Not using levelSize to separate levels

    ❌ Without level separation, can't identify "last node of level"

    ✅ Capture queue.size() BEFORE the inner loop starts

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Right/Left side view" of a tree
  • ✅ "Level-order traversal" / "Level averages" / "Zigzag traversal"
  • ✅ Any problem needing per-level processing
  • ✅ "Maximum width of binary tree"
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Max depth" / "Is balanced" — DFS is simpler for these
  • ❌ "Path sum" / "LCA" — DFS is more natural for path-based problems
  • ❌ Space-constrained and tree is very wide — BFS queue can hold O(n/2) nodes for balanced tree
