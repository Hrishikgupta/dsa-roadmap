#28 — Lowest Common Manager of Two Employees

🔖 Pattern: LCA (Lowest Common Ancestor)

🎯 Real-World Scenario START HERE ▼

Do employees ke beech conflict hai. HR ko find karna hai: "Inke common manager kaun hai jo sabse neeche (closest) hai?"

         3
        / \
       5   1
      / \ / \
     6  2 0  8
       / \
      7   4

LCA(5, 1) = 3  (root is their common ancestor)
LCA(5, 4) = 5  (5 is ancestor of 4, so 5 itself is LCA)
LCA(6, 4) = 5  (both under 5's subtree)

This is LeetCode #236 — Lowest Common Ancestor of a Binary Tree. Asked at every FAANG company.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a binary tree and two nodes p and q, find their Lowest Common Ancestor (LCA)
  • LCA = the deepest node that is an ancestor of both p and q
  • A node can be its own ancestor (if p is ancestor of q, then LCA = p)

Input → Output:

  • Input: TreeNode root, TreeNode p, TreeNode q
  • Output: TreeNode — the LCA node

Three possible cases:

  1. p and q are in different subtrees → current node is LCA
  2. p is ancestor of q → p is LCA
  3. q is ancestor of p → q is LCA

Connection to #26-27:

  • #26: DFS returns a value (height)
  • #27: DFS returns a value + validates (height or -1)
  • #28: DFS returns a node (found target or null) — same template, different return type
💪 2. Brute Force Thinking STEP 2 ▼

Approach 1: Find paths, compare — O(n) time, O(n) space

  • Find path from root to p: [3, 5]
  • Find path from root to q: [3, 5, 2, 4]
  • Compare paths — last common node = 5 = LCA

Works but needs two traversals + path storage. Can we do it in one pass?

🤔 What if DFS returns "I found p" or "I found q" or "I found both" from each subtree? The node where left finds one and right finds the other = LCA.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — DFS that returns "found" signals:

  • Each recursive call returns: the target node if found in that subtree, or null if not found
  • At each node, check left result and right result:
Left result Right result Meaning Return
not nullnot nullp in left, q in right (or vice versa)current node = LCA!
not nullnullBoth targets in left subtreeleft result (LCA is deeper)
nullnot nullBoth targets in right subtreeright result (LCA is deeper)
nullnullNeither target in this subtreenull
 
TreeNode lca(node, p, q):
    if node == null → return null
    if node == p || node == q → return node  // found a target!

    left  = lca(node.left, p, q)
    right = lca(node.right, p, q)

    if left != null && right != null → return node  // split point = LCA
    return left != null ? left : right              // propagate the found one

💡 Pattern: DFS returning found-node signals. When both sides return non-null → current node is the LCA.

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Algorithm:

  1. If node is null → return null (not found)
  2. If node is p or q → return node (found a target — don't go deeper)
  3. Recurse left and right
  4. If both non-null → this node is LCA (targets split here)
  5. If one non-null → propagate it upward (both targets are in that subtree)

💡 Step 2 is key: when we find p, we return p immediately without searching deeper. If q is below p, then p IS the LCA (a node is its own ancestor). This handles case 2 and 3 automatically.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why returning early at p/q works for the "ancestor" case:

  • If p=5 and q=4, and 4 is below 5:
  • When DFS reaches 5, it returns 5 immediately (found p)
  • It never searches below 5 for q=4
  • But that's fine — if q is below p, then p IS the LCA by definition
  • The algorithm assumes both p and q exist in the tree (guaranteed by problem)

📊 7. Complexity Analysis STEP 7 ▼
Time:O(n)
Space:O(h)

Visit each node at most once. Space = recursion depth = height of tree.

☕ 8. Java Implementation STEP 8 ▼ 
public class LowestCommonAncestor {

    public static TreeNode lowestCommonAncestor(TreeNode root,
                                                  TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) {
            return root;
        }

        TreeNode left  = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);

        if (left != null && right != null) return root; // split point
        return left != null ? left : right;             // propagate
    }
}

💡 7 lines of logic. One of the most elegant recursive solutions in all of DSA. Memorize this — it's asked constantly.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Not handling "node is its own ancestor"

    ❌ Only checking root == null, not root == p || root == q

    ✅ The base case must include: root == null || root == p || root == q

  2. Assuming p and q are always in different subtrees

    They might be in the same subtree, or one might be ancestor of the other. The algorithm handles all cases.

  3. Confusing with BST LCA (LC #235)

    BST LCA uses value comparison (go left if both < root, right if both > root). General tree LCA (#236) uses this DFS approach — no value comparison.

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Lowest Common Ancestor" / "Nearest common parent"
  • ✅ "Distance between two nodes" (= depth(p) + depth(q) - 2×depth(LCA))
  • ✅ Any problem needing to find where two paths diverge in a tree
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ BST — use value-based comparison (simpler, O(h))
  • ❌ Multiple LCA queries on same tree — preprocess with Euler tour + sparse table for O(1) per query
  • ❌ p or q might not exist in tree — need to verify existence first (this algo assumes both exist)
💡 "LCA = DFS returning found-node signals. Both sides non-null? You're the split point — you're the LCA. One side null? Propagate the other. 7 lines of pure elegance."
