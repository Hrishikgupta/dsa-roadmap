#35 — Cheapest Flight Within K Stops

🔖 Pattern: Graph BFS / Bellman-Ford Variant

🎯 Real-World Scenario START HERE ▼

Tum flight booking app bana rahe ho. User ko Delhi se New York jaana hai with at most 1 stop:

Cities: 0(Delhi), 1(Dubai), 2(London), 3(New York)
Flights: [from, to, price]
  0→1: ₹200    0→2: ₹500
  1→2: ₹100    1→3: ₹600
  2→3: ₹200

  0(Delhi)──200──→1(Dubai)──600──→3(NY)  = ₹800 (1 stop)
  0(Delhi)──500──→2(London)─200──→3(NY)  = ₹700 (1 stop) ✓
  0(Delhi)──200──→1──100──→2──200──→3    = ₹500 (2 stops) ✗ exceeds K=1

Question: "Cheapest flight from Delhi to NY with at most K=1 stop?"

Answer: ₹700 (Delhi→London→NY)

This is LeetCode #787 — Cheapest Flights Within K Stops. Combines shortest path with a constraint.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given n cities, flights with prices, source, destination, and max K stops
  • Find cheapest price from src to dst with at most K stops (K+1 edges)
  • Return -1 if no such route exists

Why standard BFS (#32) doesn't work:

  • BFS finds shortest path by NUMBER OF EDGES (unweighted)
  • Here edges have WEIGHTS (prices) — cheapest ≠ fewest edges
  • Also have a constraint: max K stops

Three approaches:

Approach Time Idea
Bellman-Ford (K+1 rounds)O(K × E)Relax all edges K+1 times
BFS with pruningO(V × K)BFS level = stops, track min cost per node per level
Dijkstra + stops trackingO(E × K log(VK))Priority queue with (cost, node, stops)

Bellman-Ford variant is the cleanest and most interview-friendly for this specific problem.

💪 2. Brute Force Thinking STEP 2 ▼

DFS: try all paths with ≤ K stops, track minimum cost.

Exponential — too many paths. Need something smarter.

🤔 Bellman-Ford relaxes ALL edges in each round. After round i, we know the shortest path using at most i edges. After K+1 rounds → shortest path with at most K stops.

⚡ 3. Optimization Thinking STEP 3 ▼

Bellman-Ford — the "relax all edges" approach:

  • Maintain dist[i] = cheapest cost to reach city i
  • Initialize: dist[src] = 0, all others = ∞
  • Repeat K+1 times (K stops = K+1 edges):
    • For each flight (u, v, price): if dist[u] + price < dist[v] → update dist[v]
  • Answer = dist[dst] (or -1 if still ∞)

Critical trick — use a COPY of dist for each round:

  • Without copy: updates in round i can cascade (use a result from the same round)
  • This would allow paths with MORE than K+1 edges
  • Solution: at start of each round, copy dist to prev. Read from prev, write to dist.
for round = 0 to K:
    prev = copy of dist
    for each flight (u, v, price):
        if prev[u] + price < dist[v]:
            dist[v] = prev[u] + price

💡 Pattern: Bellman-Ford with K+1 rounds + copy trick — guarantees at most K stops.

🎯 4. Final Approach STEP 4 ▼

Algorithm:

  1. dist = new int[n], fill with ∞, dist[src] = 0
  2. For i = 0 to K (K+1 rounds):
    • prev = Arrays.copyOf(dist)
    • For each flight [u, v, w]: if (prev[u] != ∞ && prev[u] + w < dist[v]) dist[v] = prev[u] + w
  3. Return dist[dst] == ∞ ? -1 : dist[dst]
🔍 5. Step-by-Step Dry Run STEP 5 ▼

n=4, src=0, dst=3, K=1

Flights: [0,1,200],[0,2,500],[1,2,100],[1,3,600],[2,3,200]

Round prev[] Relaxations dist[] after
Init——[0, ∞, ∞, ∞]
0 (1st edge)[0,∞,∞,∞]0→1: 200, 0→2: 500[0, 200, 500, ∞]
1 (2nd edge)[0,200,500,∞]1→2: 300, 1→3: 800, 2→3: 700[0, 200, 300, 700]

Round 1: 1→3 gives 200+600=800, but 2→3 gives 500+200=700 (cheaper). dist[3]=700.

Note: 0→1→2→3 = 500 would need 2 stops (round 2), but K=1 so we stop after round 1.

Result: dist[3] = 700 ✓ (Delhi→London→NY, 1 stop)

⚙️ 6. Internal Mechanics STEP 6 ▼

Why the copy trick is essential:

Without copy (WRONG):
Round 0: process 0→1 (dist[1]=200), then 1→3 (dist[3]=800)
But 1→3 used dist[1]=200 which was SET IN THIS ROUND
→ This is a 2-edge path (0→1→3) found in round 0 (should need round 1)

With copy (CORRECT):
Round 0: prev=[0,∞,∞,∞]. 1→3: prev[1]=∞ → skip (can't reach 1 yet)
→ Only direct flights from src are found in round 0. Correct!

Shortest path algorithms comparison:

Algorithm Weights Constraint Time Problem
BFSUnweightedNoneO(V+E)#32

Bellman-Ford (K rounds)AnyMax K edgesO(K×E)#35 (this)
Bellman-Ford (full)Any (neg ok)NoneO(V×E)Neg weights
📊 7. Complexity Analysis STEP 7 ▼
Time:O(K × E)
Space:O(V)

K+1 rounds × E edges per round. Space: two arrays of size V (dist + prev copy).

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.Arrays;

public class CheapestFlight {

    public static int findCheapestPrice(int n, int[][] flights,
                                          int src, int dst, int k) {
        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        for (int i = 0; i <= k; i++) {
            int[] prev = Arrays.copyOf(dist, n); // COPY!
            for (int[] f : flights) {
                int u = f[0], v = f[1], w = f[2];
                if (prev[u] != Integer.MAX_VALUE
                    && prev[u] + w < dist[v]) {
                    dist[v] = prev[u] + w;
                }
            }
        }

        return dist[dst] == Integer.MAX_VALUE ? -1 : dist[dst];
    }

    public static void main(String[] args) {
        int[][] flights = {{0,1,200},{0,2,500},{1,2,100},{1,3,600},{2,3,200}};
        System.out.println(findCheapestPrice(4, flights, 0, 3, 1)); // 700
    }
}

💡 The entire solution is 15 lines. The Arrays.copyOf line is the key — without it, the algorithm is wrong.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Not copying dist array each round

    ❌ Cascading updates allow paths with more than K+1 edges

    ✅ int[] prev = Arrays.copyOf(dist, n) at start of each round

  2. K+1 rounds vs K rounds

    ❌ K stops = K+1 edges. Loop should run K+1 times (0 to K inclusive).

  3. Integer overflow: prev[u] + w

    ❌ If prev[u] = MAX_VALUE, adding w overflows

    ✅ Check prev[u] != Integer.MAX_VALUE before adding

🔑 11. When to Use This Pattern STEP 11 ▼
  • ✅ "Cheapest/shortest path with at most K edges/stops"
  • ✅ Weighted graph + edge count constraint
  • ✅ Negative weights allowed (Bellman-Ford handles them, Dijkstra doesn't)
  • ✅ "Network delay time" (LC #743) — Dijkstra without K constraint
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Unweighted graph — use BFS (#32), much simpler
  • ❌ No edge constraint — use standard Dijkstra (faster)
  • ❌ Negative cycles — Bellman-Ford detects them but can't find shortest path
💡 "Cheapest path with K stops = Bellman-Ford with K+1 rounds. Copy dist array each round to prevent cascading. Read from prev, write to dist."
