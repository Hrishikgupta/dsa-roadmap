#17 — Stock Span — How Many Days Was I the King?

🔖 Pattern: Monotonic Stack (Backward Span)

🎯 Real-World Scenario START HERE ▼

Tum ek stock trading dashboard bana rahe ho. Har din ke liye ek metric dikhana hai:

"Aaj se peeche kitne consecutive days mein stock price aaj se kam ya equal tha?"

(Including today itself — "for how many days was today's price the king?")

Prices: [100, 80, 60, 70, 60, 75, 85]

Expected spans: [1, 1, 1, 2, 1, 4, 6]

  • Day 0 (100): no previous day → span = 1 (just today)
  • Day 3 (70): Day 2 (60) ≤ 70 ✓, Day 1 (80) > 70 ✗ → span = 2 (days 2,3)
  • Day 5 (75): Days 4(60),3(70),2(60) all ≤ 75, Day 1(80) > 75 → span = 4
  • Day 6 (85): Days 5(75),4(60),3(70),2(60),1(80) all ≤ 85, Day 0(100) > 85 → span = 6

This is LeetCode #901 — Online Stock Span.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • For each day, find the number of consecutive days (ending at today) where the price was ≤ today's price
  • Equivalently: find the distance to the previous day with a strictly greater price, then span = that distance
  • If no previous greater price exists → span = current index + 1 (all days from start)

Input → Output:

  • Input: int[] prices (or stream of prices one by one)
  • Output: int[] spans

#16 vs #17 — same tool, different direction:

Aspect #16 Daily Temperatures #17 Stock Span

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: For each day, scan backwards until you find a day with higher price.

// Brute Force — O(n²)
int[] span = new int[n];
for (int i = 0; i < n; i++) {
    span[i] = 1;
    int j = i - 1;
    while (j >= 0 && prices[j] <= prices[i]) {
        span[i]++;
        j--;
    }
}

O(n²) worst case (ascending prices — every day scans all the way back).

🤔 Problem #16 mein humne seekha: monotonic stack se "next greater" O(n) mein milta hai. Yahan "previous greater" chahiye — same stack, bas answer recording ka timing alag hai.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — Reusing #16's logic:

In #16, when we popped an element, the popped element got its answer (next greater = current element).

Here, after popping all smaller elements, the current element gets its answer:

  • If stack is empty after popping → no previous greater exists → span = i + 1
  • If stack has something → top of stack is the previous greater → span = i - stack.peek()

The "Jumping" Optimization (alternative view):

Instead of scanning back one-by-one, we can "jump" over already-computed spans:

  • Day 6 (85): Day 5 has span 4, meaning days 2-5 are all ≤ 75 ≤ 85
  • So we can jump from day 5 directly to day 1 (skip 4 days at once)
  • The stack does this jumping automatically — popping skips over entire groups

💡 Pattern: Monotonic Decreasing Stack — pop smaller/equal, span = distance to stack top (previous greater)

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Monotonic Decreasing Stack (Previous Greater Element)

Algorithm:

  1. Create empty stack (stores indices)
  2. For i = 0 to n - 1:
    • While stack not empty AND prices[stack.peek()] <= prices[i]: pop
    • If stack empty: span[i] = i + 1 (no previous greater)
    • Else: span[i] = i - stack.peek() (distance to previous greater)
    • Push i

💡 Note: we use <= (not <) because span includes days with equal price. We pop equal elements too.

⚙️ 6. Internal Mechanics STEP 6 ▼

#16 vs #17 — the subtle difference:

Step #16 (Next Greater) #17 (Previous Greater / Span)
Pop conditioncurrent > stack topcurrent >= stack top (includes equal)
Who gets the answer?The popped elementThe current element
Answer formularesult[popped] = i - poppedspan[i] = i - stack.peek() (or i+1 if empty)

Online vs Batch processing:

  • LC #901 is an "online" problem — prices arrive one at a time via next(price)
  • Stack persists across calls — each call pushes one element and pops as needed
  • Our batch version processes all at once, but the logic is identical

☕ 8. Java Implementation STEP 8 ▼

Batch version (all prices at once):

import java.util.ArrayDeque;
import java.util.Arrays;
import java.util.Deque;

public class StockSpan {

    public static int[] calculateSpan(int[] prices) {
        int n = prices.length;
        int[] span = new int[n];
        Deque<Integer> stack = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && prices[stack.peek()] <= prices[i]) {
                stack.pop();
            }
            span[i] = stack.isEmpty() ? i + 1 : i - stack.peek();
            stack.push(i);
        }
        return span;
    }

    public static void main(String[] args) {
        int[] prices = {100, 80, 60, 70, 60, 75, 85};
        System.out.println(Arrays.toString(calculateSpan(prices)));
        // [1, 1, 1, 2, 1, 4, 6]
    }
}

Online version (LC #901 — prices arrive one at a time):

class StockSpanner {
    Deque<int[]> stack; // [price, span]

    public StockSpanner() {
        stack = new ArrayDeque<>();
    }

    public int next(int price) {
        int span = 1;
        while (!stack.isEmpty() && stack.peek()[0] <= price) {
            span += stack.pop()[1]; // absorb popped element's span
        }
        stack.push(new int[]{price, span});
        return span;
    }
}

💡 Online version stores [price, span] pairs. When popping, we absorb the popped element's span into ours — that's the "jumping" optimization.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Using < instead of <= in pop condition

    ❌ prices[stack.peek()] < prices[i] → equal prices don't get popped → wrong span for equal days

    ✅ prices[stack.peek()] <= prices[i] → equal prices are part of the span

  2. Confusing who gets the answer (#16 vs #17)

    #16: popped element gets answer. #17: current element gets answer after popping.

  3. Forgetting the empty stack case

    ❌ span[i] = i - stack.peek() when stack is empty → exception

    ✅ Check stack.isEmpty() first → span = i + 1

  • ✅ "Consecutive count looking backward"
  • ✅ Online/streaming data where you process one element at a time
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Maximum profit from buying/selling" — that's Problem #1 (tracking variable), not stack
  • ❌ "K-day moving average" — that's sliding window (#8)
  • ❌ Non-consecutive span (e.g., "total days with lower price, not necessarily consecutive")
💡 "Stock Span = Previous Greater Element. Same monotonic decreasing stack as #16, but the CURRENT element gets the answer (distance to stack top), not the popped one."
