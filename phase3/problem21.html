#21 — Generate All Subsets of a Menu

🔖 Pattern: Recursion (Include/Exclude)

🎯 Real-World Scenario START HERE ▼

Tum ek restaurant app bana rahe ho. Menu mein 3 items hain: [Pizza, Burger, Pasta]

Customer ko saare possible meal combos dikhane hain — koi bhi subset of items choose kar sakta hai (including nothing).

All subsets:

[]                    ← nothing (empty set)
[Pizza]               ← just one item
[Burger]
[Pasta]
[Pizza, Burger]       ← two items
[Pizza, Pasta]
[Burger, Pasta]
[Pizza, Burger, Pasta] ← all items

Total: 2³ = 8 subsets for 3 items.

This is LeetCode #78 — Subsets. Foundation for backtracking and DP.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an array of distinct integers, return all possible subsets (the power set)
  • Include the empty set
  • No duplicate subsets

Input → Output:

  • Input: int[] nums (distinct elements)
  • Output: List<List<Integer>> — all subsets

The key insight — binary decision tree:

For each element, you have exactly two choices: include it or exclude it.

                    []
                 /      \
          [1]              []          ← include 1 or not?
         /    \          /    \
     [1,2]    [1]     [2]     []      ← include 2 or not?
     / \      / \     / \     / \
 [1,2,3][1,2][1,3][1][2,3][2][3][]   ← include 3 or not?

Leaves of this tree = all 2ⁿ subsets. Each path from root to leaf = a sequence of include/exclude decisions.

💪 2. Brute Force Thinking STEP 2 ▼

Iterative approach (bit manipulation):

  • For n elements, there are 2ⁿ subsets
  • Each subset can be represented by an n-bit number (1 = include, 0 = exclude)
  • For [1,2,3]: 000=[], 001=[3], 010=[2], 011=[2,3], 100=[1], 101=[1,3], 110=[1,2], 111=[1,2,3]
// Iterative — O(n × 2ⁿ)
for (int mask = 0; mask < (1 << n); mask++) {
    List<Integer> subset = new ArrayList<>();
    for (int i = 0; i < n; i++) {
        if ((mask & (1 << i)) != 0) subset.add(nums[i]);
    }
    result.add(subset);
}

Works, but doesn't teach recursive thinking. The recursive approach is more important for interviews because it generalizes to backtracking.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Buffet Line Analogy:

You're walking through a buffet. At each dish station, you decide: take it or skip it.

  • Station 1 (Pizza): Take → [Pizza] path. Skip → [] path.
  • Station 2 (Burger): For EACH existing path, again take or skip.
  • Station 3 (Pasta): Same — take or skip for each path.
  • After the last station, every path you took = one unique subset.

Recursive structure:

generateSubsets(index, currentSubset):
    if index == n:           // Base: processed all elements
        add currentSubset to result
        return

    // Choice 1: INCLUDE nums[index]
    currentSubset.add(nums[index])
    generateSubsets(index + 1, currentSubset)
    currentSubset.removeLast()   // BACKTRACK (undo the choice)

    // Choice 2: EXCLUDE nums[index]
    generateSubsets(index + 1, currentSubset)

⚠️ The backtrack step is critical:

  • After exploring the "include" branch, we must UNDO the addition before exploring "exclude"
  • This is called backtracking — undo your choice to try the other option
  • Without it, the "exclude" branch would still have the element in currentSubset

💡 Pattern: Include/Exclude recursion with backtracking — the foundation of all subset/combination problems

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Recursion — Include/Exclude (Binary Decision Tree)

Algorithm:

  1. Start with empty current list and index = 0
  2. At each index, make two recursive calls:
    • Include nums[index] → recurse with index+1
    • Exclude (backtrack) → recurse with index+1
  3. When index == n → add a copy of current to result

💡 Must add a COPY (new ArrayList<>(current)), not the reference. Otherwise all entries in result point to the same (eventually empty) list.

🔍 5. Step-by-Step Dry Run STEP 5 ▼

Input: nums = [1, 2, 3]

Call tree (I = include, E = exclude):

                        f(0, [])
                    /                \
            I: f(1, [1])           E: f(1, [])
            /          \            /          \
     I: f(2,[1,2])  E: f(2,[1])  I: f(2,[2])  E: f(2,[])
      /      \      /     \     /     \     /     \
  [1,2,3] [1,2] [1,3] [1]  [2,3] [2]  [3]  []

Execution order (DFS — left branch first):

# Action current Added to result?
1Include 1, Include 2, Include 3 → base[1,2,3]✓
2Backtrack 3, Exclude 3 → base[1,2]✓
3Backtrack 2, Exclude 2, Include 3 → base[1,3]✓
4Backtrack 3, Exclude 3 → base[1]✓
5-8Same pattern for Exclude 1 branch[2,3],[2],[3],[]✓✓✓✓

Result: [[1,2,3],[1,2],[1,3],[1],[2,3],[2],[3],[]] — all 8 subsets ✓

⚙️ 6. Internal Mechanics STEP 6 ▼

Why backtracking is essential:

// WITHOUT backtrack (WRONG):
current = [1]
include 2 → current = [1, 2]
// Now explore "exclude 2" branch...
// But current is STILL [1, 2]! Should be [1]!

// WITH backtrack (CORRECT):
current = [1]
include 2 → current = [1, 2]
  // ... explore include branch ...
current.removeLast() → current = [1]  // UNDO!
// Now explore "exclude 2" with correct state [1]

Include/Exclude vs other subset generation methods:

Method Approach Best for
Include/ExcludeBinary tree — take or skip each elementSubsets, DP foundation
CascadingStart with [[]], add each element to all existing subsetsIterative, easy to understand
Bit manipulationEach number 0 to 2ⁿ-1 = a subsetWhen n ≤ 20

Include/Exclude is the most important because it directly maps to backtracking (#36-38) and DP (#41, #43).

📊 7. Complexity Analysis STEP 7 ▼
Time: O(n × 2ⁿ)
Space: O(n)

Time: 2ⁿ subsets, each takes O(n) to copy into result.

Space: O(n) recursion depth + O(n) for current list. Output space O(n × 2ⁿ) not counted.

This is optimal — you MUST generate 2ⁿ subsets, so O(2ⁿ) is unavoidable.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.*;

public class Subsets {

    public static List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        generate(nums, 0, new ArrayList<>(), result);
        return result;
    }

    private static void generate(int[] nums, int idx,
                                  List<Integer> current,
                                  List<List<Integer>> result) {
        if (idx == nums.length) {
            result.add(new ArrayList<>(current)); // add COPY
            return;
        }

        // Include nums[idx]
        current.add(nums[idx]);
        generate(nums, idx + 1, current, result);
        current.remove(current.size() - 1); // BACKTRACK

        // Exclude nums[idx]
        generate(nums, idx + 1, current, result);
    }

    public static void main(String[] args) {
        System.out.println(subsets(new int[]{1, 2, 3}));
        // [[1,2,3],[1,2],[1,3],[1],[2,3],[2],[3],[]]
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Adding reference instead of copy to result

    ❌ result.add(current) → all entries point to same list (ends up all empty)

    ✅ result.add(new ArrayList<>(current)) → snapshot copy

  2. Forgetting to backtrack

    ❌ No current.remove() after include branch → exclude branch has wrong state

    ✅ Always undo: current.remove(current.size() - 1)

  3. Using wrong index for removal

    ❌ current.remove(idx) → removes by index in the ORIGINAL array, not the current list

    ✅ current.remove(current.size() - 1) → always remove the last added element

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify):

  • ✅ "Generate all subsets" / "Power set"
  • ✅ "All combinations" / "All possible selections"
  • ✅ "For each element, choose to include or not"
  • ✅ "2ⁿ possibilities" — binary decision at each step

This pattern evolves into:

  • #22: Permutations — instead of include/exclude, try each remaining element at each position
  • #36-38: Backtracking — include/exclude with constraints (pruning)
  • #41, #43: DP — include/exclude with optimization (memoization)
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Find ONE subset with sum = target" — use DP, not generate all subsets
  • ❌ n > 20 — 2²⁰ ≈ 1M is feasible, 2³⁰ ≈ 1B is not
  • ❌ "Permutations" — different pattern (order matters, #22)
  • ❌ "Combinations of size k" — use k-limited recursion, not full power set
💡 "Subsets = binary decision tree. Each element: include or exclude. Backtrack after include branch. 2ⁿ leaves = 2ⁿ subsets."
