#16 — Next Warmer Day Forecast

🔖 Pattern: Monotonic Stack

🎯 Real-World Scenario START HERE ▼

Tum ek weather app bana rahe ho. User ko har din ke liye batana hai:

"Kitne din baad agle warmer day aayega?"

Temperatures: [73, 74, 75, 71, 69, 72, 76, 73]

Expected output: [1, 1, 4, 2, 1, 1, 0, 0]

  • Day 0 (73°): next warmer = Day 1 (74°) → 1 day
  • Day 2 (75°): next warmer = Day 6 (76°) → 4 days
  • Day 6 (76°): no warmer day after → 0

This is LeetCode #739 — Daily Temperatures. The classic Monotonic Stack introduction.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • For each element, find the distance to the next greater element to its right
  • If no greater element exists, answer is 0

Input → Output:

  • Input: int[] temperatures
  • Output: int[] result — days until warmer temperature (0 if none)

This is the "Next Greater Element" (NGE) family:

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: For each day, scan right until you find a warmer day.

// Brute Force — O(n²)
int[] result = new int[n];
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (temps[j] > temps[i]) {
            result[i] = j - i;
            break;
        }
    }
}

O(n²) worst case (descending temperatures — every day scans to the end).

🤔 Observation: Jab hum right se left process karein, toh ek stack mein "waiting" elements rakh sakte hain. Jab koi bada element aaye, toh saare chhote elements ka answer mil jaata hai ek saath.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Queue at a Ticket Counter Analogy:

Imagine people standing in a line, each holding a sign with their temperature. A new person arrives:

  • If the new person is taller (warmer) than people at the back of the line:
  • Those shorter people say: "Found my answer! This person is my next warmer day." They leave.
  • The new person joins the back of the line and waits.

The line always stays in decreasing order (tallest at front, shortest at back). That's a Monotonic Decreasing Stack.

What is a Monotonic Stack?

  • A stack where elements are always in sorted order (increasing or decreasing)
  • When a new element violates the order → pop elements until order is restored
  • Each popped element gets its answer (the new element is its "next greater/smaller")

For "Next Greater Element":

  • Maintain a decreasing stack (top = smallest waiting element)
  • New element arrives → pop all elements smaller than it (they found their answer)
  • Push new element onto stack (it's now waiting for ITS next greater)

💡 Pattern: Monotonic Decreasing Stack — pop smaller elements when a bigger one arrives

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Monotonic Decreasing Stack (stores indices)

Algorithm:

  1. Create empty stack (stores indices, not values)
  2. Create result array initialized to 0
  3. For i = 0 to n - 1:
    • While stack not empty AND temps[i] > temps[stack.peek()]:
      • prevIndex = stack.pop()
      • result[prevIndex] = i - prevIndex (distance to next warmer)
    • stack.push(i)
  4. Remaining elements in stack → no warmer day → result stays 0

💡 We store INDICES in the stack (not values) because we need to calculate distance. We look up values via temps[stack.peek()].

⚙️ 6. Internal Mechanics STEP 6 ▼

Why it's called "Monotonic":

  • The stack always maintains a monotonic order (strictly decreasing in this case)
  • New element either fits the order (push) or breaks it (pop until order restored, then push)
  • At any point, stack bottom → top = decreasing temperatures

Why it's O(n) despite the inner while loop:

  • Each element is pushed exactly once and popped at most once
  • Total pushes = n, total pops ≤ n
  • Total operations = 2n = O(n)
  • Same amortized argument as variable sliding window (#9)

Time: O(n)

Each element pushed once, popped at most once = O(2n) = O(n). Stack holds at most n elements.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.ArrayDeque;
import java.util.Arrays;
import java.util.Deque;

public class DailyTemperatures {

    public static int[] dailyTemperatures(int[] temps) {
        int n = temps.length;
        int[] result = new int[n];
        Deque<Integer> stack = new ArrayDeque<>(); // stores indices

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && temps[i] > temps[stack.peek()]) {
                int prevIdx = stack.pop();
                result[prevIdx] = i - prevIdx;
            }
            stack.push(i);
        }

        return result; // remaining indices stay 0
    }

    public static void main(String[] args) {
        int[] temps = {73,74,75,71,69,72,76,73};
        System.out.println(Arrays.toString(dailyTemperatures(temps)));
        // [1, 1, 4, 2, 1, 1, 0, 0]
    }
}

💡 Core logic is just 5 lines inside the for loop. Monotonic stack problems have beautifully concise code.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Storing values instead of indices in the stack

    ❌ Can't calculate distance without knowing the index

    ✅ Always store indices. Look up values via temps[stack.peek()]

  2. Using >= instead of >

    ❌ "Next warmer" means strictly greater. Equal temperature is NOT warmer.

    ✅ Use temps[i] > temps[stack.peek()] (strict)

  3. Forgetting that remaining stack elements get 0

    ✅ Initialize result array to 0. Elements that stay in stack never get updated → naturally 0.

  4. Processing right-to-left instead of left-to-right

    Both work, but left-to-right is more intuitive: "I arrive, I resolve everyone waiting who is smaller than me."

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify):

  • ✅ "Next greater element" / "Next smaller element"
  • ✅ "Previous greater/smaller element"
  • ✅ "How many days until X happens"
  • ✅ "Stock span" / "Stock price" problems
  • ✅ "Largest rectangle in histogram"
  • ✅ Any problem where each element needs to find the nearest element satisfying a comparison

Quick decision:

Need "next greater to the right"?
→ Monotonic Decreasing Stack, scan left→right

Need "next smaller to the right"?
→ Monotonic Increasing Stack, scan left→right

Need "previous greater to the left"?
→ Monotonic Decreasing Stack, scan left→right (answer = stack top after push)
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "K-th nearest greater element" — monotonic stack finds the 1st, not k-th
  • ❌ "Global maximum/minimum" — just scan once, no stack needed
  • ❌ "Sliding window maximum" — needs a Monotonic Deque (double-ended), not just stack
💡 "Next Greater Element = Monotonic Decreasing Stack. New element aaya jo bada hai? Pop all smaller ones — unka answer mil gaya."
