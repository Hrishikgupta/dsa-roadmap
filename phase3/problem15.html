#15 — Validating Nested Code Brackets

🔖 Pattern: Stack (Matching Pairs)

🎯 Real-World Scenario START HERE ▼

Tum ek code editor bana rahe ho (like VS Code). User code likhta hai with brackets:

// Valid examples:
"()"        → ✓
"()[]{}"    → ✓
"{[()]}"    → ✓  (nested)
"({[]})"   → ✓  (deeply nested)

// Invalid examples:
"(]"        → ✗  (wrong type)
"([)]"      → ✗  (wrong order — interleaved)
"((("       → ✗  (unclosed)
"}"         → ✗  (no opener)

Editor ko real-time batana hai: "Are all brackets properly matched and nested?"

This is LeetCode #20 — Valid Parentheses. The gateway problem to understanding Stacks.

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given a string containing only (){}[], determine if it's valid
  • Valid means: every opening bracket has a matching closing bracket of the same type, in the correct order

Rules for validity:

  1. Every opening bracket must be closed by the same type: (→), {→}, [→]
  2. Brackets must close in the correct order: innermost first (LIFO — Last In, First Out)
  3. Every closing bracket must have a corresponding opening bracket

Why is this a NEW paradigm?

  • Phase 1-2: Arrays, pointers, windows — all about sequential processing
  • Phase 3 starts with Stack — a data structure that processes things in reverse order (LIFO)
  • Brackets are inherently LIFO: the LAST opened bracket must be the FIRST one closed

💡 {[()]} — the ( was opened last, so it must close first with ). That's LIFO = Stack.

💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Repeatedly find and remove matching adjacent pairs until nothing is left.

// Brute Force — O(n²)
while (s.contains("()") || s.contains("[]") || s.contains("{}")) {
    s = s.replace("()", "");
    s = s.replace("[]", "");
    s = s.replace("{}", "");
}
return s.isEmpty();

Works! But O(n²) — each replace scans the string, and we might need n/2 rounds.

🤔 The key observation: when we see a closing bracket, we need to check what was the MOST RECENT unmatched opening bracket. That's exactly what a Stack does — peek at the top.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Stack of Plates Analogy:

Imagine stacking plates. You can only add to the top and remove from the top.

  • See ( → push a plate labeled "(" on top
  • See [ → push a plate labeled "[" on top
  • See ] → check top plate. Is it "["? Yes → match! Pop it. No → invalid.
  • See ) → check top plate. Is it "("? Yes → match! Pop it. No → invalid.

At the end, if the stack is empty → all brackets matched. If not → some brackets are unclosed.

What is a Stack?

  • LIFO — Last In, First Out
  • push(x) — add to top: O(1)
  • pop() — remove from top: O(1)
  • peek() — look at top without removing: O(1)
  • isEmpty() — check if empty: O(1)
  • In Java: Deque<Character> stack = new ArrayDeque<>()

💡 Use ArrayDeque not Stack class in Java. The Stack class is legacy, synchronized, and slower.

💡 Pattern: Stack for matching pairs — push openers, pop and match on closers

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern: Stack — Push Openers, Match Closers

Algorithm:

  1. Create an empty stack
  2. For each character in string:
    • If it's an opening bracket ( [ { → push onto stack
    • If it's a closing bracket ) ] }:
      • If stack is empty → no matching opener → return false
      • Pop the top. If it doesn't match the closer → return false
  3. After processing all chars: return stack.isEmpty()

💡 Trick: Instead of pushing the opener and checking later, push the EXPECTED closer. Then on a closer, just check if top == current char. Cleaner code.

🔍 5. Step-by-Step Dry Run STEP 5 ▼

Input: s = "{[()]}"

Step Char Action Stack (top → bottom)
1{Push }[}]
2[Push ][], }]
3(Push )[), ], }]
4)Pop → ) == ) ✓[], }]
5]Pop → ] == ] ✓[}]
6}Pop → } == } ✓[] (empty)

Stack empty → true (valid)

Invalid example: s = "([)]"

Step Char Action Stack

false — ] was expected but ) came. Interleaved brackets are invalid.

⚙️ 6. Internal Mechanics STEP 6 ▼

Why Stack is the perfect data structure here:

  • Brackets are nested — innermost must close first
  • That's LIFO: the Last opened bracket is the First one that needs closing
  • Stack naturally maintains this order — top = most recent unmatched opener

The "push expected closer" trick:

  • Instead of: push (, then later check if ) matches ( (needs a mapping)
  • Do: push ) when you see (. Then on closer, just check stack.pop() == currentChar
  • Eliminates the need for a separate mapping lookup during the matching step

Three failure modes:

Failure Example Detection
Type mismatch(]Pop ≠ current closer
Extra closer}Stack empty when closer arrives
Unclosed opener((Stack not empty at the end
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n)
Space: O(n)

Each character is pushed at most once and popped at most once → O(n). Stack can hold at most n/2 openers → O(n) space.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.ArrayDeque;
import java.util.Deque;

public class ValidParentheses {

    public static boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();

        for (char c : s.toCharArray()) {
            if (c == '(') stack.push(')');
            else if (c == '[') stack.push(']');
            else if (c == '{') stack.push('}');
            else {
                if (stack.isEmpty() || stack.pop() != c) {
                    return false;
                }
            }
        }

        return stack.isEmpty();
    }

    public static void main(String[] args) {
        System.out.println(isValid("{[()]}"));  // true
        System.out.println(isValid("([)]"));   // false
        System.out.println(isValid("(("));     // false
    }
}

💡 The "push expected closer" trick makes the matching logic a single stack.pop() != c check. No HashMap needed for mapping.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting to check stack.isEmpty() before popping

    ❌ stack.pop() on empty stack → EmptyStackException

    ✅ Always check stack.isEmpty() first when a closer arrives

  2. Forgetting the final stack.isEmpty() check

    ❌ Input "((" → no mismatches during scan, but stack has 2 unmatched openers

    ✅ Return stack.isEmpty() at the end, not just true

  3. Using Java's Stack class

    ⚠️ java.util.Stack is legacy, extends Vector (synchronized = slow)

    ✅ Use Deque<Character> stack = new ArrayDeque<>()

  4. Only counting brackets instead of matching types

    ❌ Counting ( and ) separately → "(]" would pass (1 open, 1 close)

    ✅ Must match TYPE, not just count

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify):

  • ✅ "Valid parentheses" / "Balanced brackets"
  • ✅ "Matching pairs" — anything that opens and closes
  • ✅ "Nested structure" — HTML tags, function calls, expressions
  • ✅ "Most recent unmatched" — LIFO = Stack
  • ✅ "Undo" operations — last action undone first

🚫 12. When NOT to Use STEP 12 ▼
  • ❌ "Minimum number of bracket removals to make valid" — needs counting, not just stack
  • ❌ "Generate all valid bracket combinations" — that's recursion/backtracking (#21-23)
  • ❌ FIFO order needed — use Queue instead of Stack
💡 "Jab bhi 'most recent unmatched' dhundhna ho — Stack lagao. Push openers, pop and match closers. End mein stack empty = valid."
