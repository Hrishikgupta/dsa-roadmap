#1 — Finding the Best Sale Day (Max Profit)

🔖 Pattern: Linear Scan / Tracking Variables

🎯 Real-World Scenario START HERE ▼

Tum ek shopkeeper ho. Tumhare paas last 7 days ke product prices hain:

[7, 1, 5, 3, 6, 4]

Tum ek hi baar buy kar sakte ho aur ek hi baar sell — lekin buy pehle hona chahiye sell se.

Question: Maximum kitna profit ho sakta hai?

Answer: Buy at ₹1 (day 2), Sell at ₹6 (day 5) → Profit = ₹5

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an array of prices (each index = a day)
  • Find the maximum difference prices[j] - prices[i] where j > i
  • If no profit possible, return 0

Input → Output Clarity:

  • Input: int[] prices — array of daily prices
  • Output: int — maximum profit (or 0)

Constraints to note:

  • Buy MUST happen before sell (left to right order matters)
  • Only ONE transaction allowed (one buy + one sell)
  • Prices are non-negative
💪 2. Brute Force Thinking STEP 2 ▼

First instinct: Try every possible pair (buy day, sell day)

  • For each day i (buy), check every day j > i (sell)
  • Calculate prices[j] - prices[i]
  • Track the maximum
// Brute Force — O(n²)
int maxProfit = 0;
for (int i = 0; i < prices.length; i++) {
    for (int j = i + 1; j < prices.length; j++) {
        int profit = prices[j] - prices[i];
        maxProfit = Math.max(maxProfit, profit);
    }
}
return maxProfit;

Problem: Two nested loops → O(n²). For n = 100,000 that's 10 billion operations. TLE guaranteed.

🤔 Sochne ka point: Kya hume sach mein har pair check karna zaroori hai?

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight: Jab tum left-to-right traverse kar rahe ho, tumhe sirf ek cheez track karni hai:

  • Ab tak ka minimum price (best day to buy so far)

Har day pe tum pooch rahe ho:

  • "Agar main aaj sell karun, aur maine ab tak ke sabse saste din pe buy kiya hota, toh kitna profit hota?"
  • profit = today's price - minimum price so far

Why does this work?

  • Minimum price hamesha left side pe hoga (past mein) — so buy-before-sell condition automatically satisfied
  • Har element ko sirf ek baar dekhna padta hai → O(n)

💡 Pattern identified: Linear Scan with Tracking Variable

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern Name: Linear Scan / Tracking Variables

Why this pattern fits:

  • We need to find a relationship between current element and something from the past
  • Instead of looking back (nested loop), we carry the past forward in a variable
  • One variable (minPrice) replaces an entire inner loop

Algorithm:

  1. Initialize minPrice = Integer.MAX_VALUE
  2. Initialize maxProfit = 0
  3. For each price in array:
    • If price < minPrice → update minPrice
    • Else → calculate profit, update maxProfit if bigger
  4. Return maxProfit
🔍 5. Step-by-Step Dry Run STEP 5 ▼

Input: [7, 1, 5, 3, 6, 4]

Day Price minPrice Profit Today maxProfit Action
077 ✓—0New min found
111 ✓—0New min found
2515-1 = 44 ✓Profit updated
3313-1 = 24No update
4616-1 = 55 ✓Profit updated
5414-1 = 35No update

Result: maxProfit = 5 (Buy at 1, Sell at 6)

⚙️ 6. Internal Mechanics STEP 6 ▼

Behind the scenes — why this is O(n):

  • Single pass: pointer moves left → right, never goes back
  • minPrice acts as a "memory" of the best buy opportunity seen so far
  • At each step, we make a greedy decision: "Is today's price a new minimum? Or can I profit from it?"

Why we don't need to look back:

  • If a new minimum appears at index k, any future sell will always prefer this minimum over older ones
  • The old minimum is "dominated" — it can never produce a better result than the new one for future sells
  • This is the core idea of tracking variables: compress the past into a single value
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n)
Space: O(1)

Comparison with Brute Force:

Approach Time Space n = 100,000
Brute ForceO(n²)O(1)10,000,000,000 ops
OptimizedO(n)O(1)100,000 ops

100,000x faster. Yahi hai optimization ka power.

☕ 8. Java Implementation STEP 8 ▼ 
public class BestTimeToBuySellStock {

    public static int maxProfit(int[] prices) {
        int minPrice = Integer.MAX_VALUE;
        int maxProfit = 0;

        for (int price : prices) {
            if (price < minPrice) {
                minPrice = price;
            } else {
                maxProfit = Math.max(maxProfit, price - minPrice);
            }
        }
        return maxProfit;
    }

    public static void main(String[] args) {
        int[] prices = {7, 1, 5, 3, 6, 4};
        System.out.println("Max Profit: " + maxProfit(prices)); // Output: 5
    }
}

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Finding global max and global min separately

    ❌ [3, 8, 1, 2] → min=1, max=8, profit=7? WRONG. Min comes AFTER max here.

    ✅ Correct answer is 5 (buy at 3, sell at 8)

  2. Not handling the "no profit" case

    ❌ Returning negative profit instead of 0

    ✅ Initialize maxProfit = 0, so it naturally handles this

  3. Using if-else instead of both checks

    Note: Using else is fine here because if price IS the new minimum, selling on the same day gives 0 profit anyway. Both approaches work.

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify) this pattern:

  • ✅ "Find maximum/minimum difference in array"
  • ✅ "One pass through data is enough"
  • ✅ Current element depends on something from the LEFT side only
  • ✅ You need to track a running min/max/sum
  • ✅ Problem says "buy before sell" or any ordering constraint

Real-world uses:

  • Stock trading algorithms
  • Finding best time to buy/sell anything
  • Temperature difference problems
  • Any "best pair with ordering" problem
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ Multiple transactions allowed (that's a different DP problem)
  • ❌ You need to find the actual pair of indices (need slight modification)
  • ❌ Array is unsorted and you need closest pair (different pattern)
  • ❌ Problem requires looking at BOTH left and right context (need prefix/suffix arrays)
💡 "Jab past ka sirf ek value chahiye — use a tracking variable, not a nested loop."
