#4 — Rotating the Security Shift Schedule

🔖 Pattern: Array Rotation (Reversal Trick)

🎯 Real-World Scenario START HERE ▼

Tum ek security company ke manager ho. 7 guards hain weekly rotation mein:

[A, B, C, D, E, F, G] (Monday → Sunday)

Boss bola: "Schedule ko 3 positions right rotate karo" — taaki last 3 guards pehle aa jayein.

Expected result: [E, F, G, A, B, C, D]

Constraint: Extra array mat banao — in-place karo, O(1) space mein.

💡 Bina extra array ke array ko rotate kaise karein?

🧠 1. Problem Understanding STEP 1 ▼

What exactly is being asked?

  • Given an array and a number k, rotate the array to the right by k steps
  • Right rotation by 1: last element goes to front, everything else shifts right
  • Right rotation by k: do this k times (or find a smarter way)

Input → Output:

  • Input: int[] nums, int k
  • Output: modify nums in-place (no return value)

Key observation:

  • If k = array length, array comes back to original → k = k % n
  • Right rotate by k = last k elements move to front, first n-k elements move to back

Connection to previous problems:

  • #1, #2, #3 — we used extra space (variables, HashMap, prefix array) to optimize time
  • #4 — now the challenge is O(1) space. How to manipulate array without extra memory?
💪 2. Brute Force Thinking STEP 2 ▼

Approach 1: Rotate one step at a time — O(n × k)

// Rotate right by 1, k times
for (int i = 0; i < k; i++) {
    int last = nums[nums.length - 1];
    for (int j = nums.length - 1; j > 0; j--) {
        nums[j] = nums[j - 1];
    }
    nums[0] = last;
}

Time: O(n × k). If k ≈ n → O(n²). Space: O(1) ✓ but too slow.

Approach 2: Use extra array — O(n) time, O(n) space

int[] temp = new int[n];
for (int i = 0; i < n; i++) {
    temp[(i + k) % n] = nums[i];
}
System.arraycopy(temp, 0, nums, 0, n);

Time: O(n) ✓ but Space: O(n) ✗. Boss said no extra array.

🤔 Can we get O(n) time AND O(1) space? That's where the reversal trick comes in.

⚡ 3. Optimization Thinking STEP 3 ▼

Key Insight — The Book Analogy:

Imagine a book with pages [1, 2, 3, 4, 5, 6, 7]. You want last 3 pages (5,6,7) to come first.

  1. Flip the first part (1,2,3,4) → becomes (4,3,2,1)
  2. Flip the second part (5,6,7) → becomes (7,6,5)
  3. Now flip the ENTIRE thing: (4,3,2,1,7,6,5) → (5,6,7,1,2,3,4) ✓

Wait — that gives left rotation. For right rotation, reverse order:

  1. Reverse entire array
  2. Reverse first k elements
  3. Reverse remaining n-k elements

Why does this work?

  • Reversing the whole array puts the last k elements at the front — but in wrong order
  • Reversing the first k fixes their internal order
  • Reversing the remaining n-k fixes their internal order too
  • Three reverses = correct rotation. Each reverse is O(n) with O(1) space (just swaps).

💡 Pattern identified: Reversal Trick — three reverses to rotate in-place

🎯 4. Final Approach (Pattern-Based) STEP 4 ▼

Pattern Name: Array Rotation via Reversal Trick

Why this pattern fits:

  • Need to rearrange array in-place with O(1) extra space
  • Reverse is the simplest O(n), O(1) operation on arrays — just swap from both ends
  • Three strategic reverses achieve any rotation

Algorithm (Right rotate by k):

  1. k = k % n (handle k > n)
  2. Reverse entire array: reverse(0, n-1)
  3. Reverse first k: reverse(0, k-1)
  4. Reverse rest: reverse(k, n-1)
🔍 5. Step-by-Step Dry Run STEP 5 ▼

Input: nums = [1, 2, 3, 4, 5, 6, 7], k = 3

k = 3 % 7 = 3

Step Operation Array State
Original — [1, 2, 3, 4, 5, 6, 7]
1 Reverse ALL (0 to 6) [7, 6, 5, 4, 3, 2, 1]
2 Reverse first k=3 (0 to 2) [5, 6, 7, 4, 3, 2, 1]
3 Reverse rest (3 to 6) [5, 6, 7, 1, 2, 3, 4] ✓

How each reverse works internally (Step 1 example):

Swap Left idx Right idx Array After
106[7, 2, 3, 4, 5, 6, 1]
215[7, 6, 3, 4, 5, 2, 1]
324[7, 6, 5, 4, 3, 2, 1] ✓

Two pointers move inward, swapping as they go. O(n/2) swaps = O(n). O(1) space (just one temp variable).

⚙️ 6. Internal Mechanics STEP 6 ▼

Why three reverses = rotation? (Mathematical proof)

Think of the array as two parts: A = [1,2,3,4] and B = [5,6,7]

  • We want: B + A = [5,6,7,1,2,3,4]
  • Step 1: Reverse whole = (AB)ʳ = Bʳ Aʳ = [7,6,5,4,3,2,1]
  • Step 2: Reverse first k = (Bʳ)ʳ Aʳ = B Aʳ = [5,6,7,4,3,2,1]
  • Step 3: Reverse rest = B (Aʳ)ʳ = B A = [5,6,7,1,2,3,4] ✓

Reverse of reverse = original. That's the entire trick.

Why k = k % n?

  • Rotating by n = full circle = back to original
  • Rotating by n+3 = same as rotating by 3
  • So effective rotation = k % n
  • Without this: if k = 1,000,000 and n = 7, brute force does 1M rotations. With mod: just 6.
📊 7. Complexity Analysis STEP 7 ▼
Time: O(n)
Space: O(1)

Breakdown: 3 reverses × O(n) each = O(3n) = O(n). Only one temp variable for swapping.

Approach Time Space
One-by-one rotationO(n × k)O(1)
Extra arrayO(n)O(n)
Reversal TrickO(n)O(1)

Best of both worlds — fast AND memory-efficient.

☕ 8. Java Implementation STEP 8 ▼ 
import java.util.Arrays;

public class RotateArray {

    public static void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n;
        if (k == 0) return;

        reverse(nums, 0, n - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, n - 1);
    }

    private static void reverse(int[] nums, int left, int right) {
        while (left < right) {
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }

    public static void main(String[] args) {
        int[] nums = {1, 2, 3, 4, 5, 6, 7};
        rotate(nums, 3);
        System.out.println(Arrays.toString(nums)); // [5, 6, 7, 1, 2, 3, 4]
    }
}

💡 The reverse() helper is reusable — you'll use it in many array problems. Keep it in your toolkit.

⚠️ 10. Common Mistakes STEP 10 ▼
  1. Forgetting k = k % n

    ❌ If k = 1,000,000 and n = 7, reversing with k=1M causes ArrayIndexOutOfBounds

    ✅ Always normalize: k = k % n

  2. Confusing left rotation with right rotation

    Right rotate by k: reverse all → reverse first k → reverse rest

    Left rotate by k: reverse first k → reverse rest → reverse all (or just right rotate by n-k)

  3. Wrong reverse boundaries

    ❌ reverse(0, k) instead of reverse(0, k-1)

    ✅ First k elements = indices 0 to k-1. Remaining = indices k to n-1.

  4. Not handling k = 0 or empty array

    ✅ Early return if k == 0 or nums.length <= 1

🔑 11. When to Use This Pattern STEP 11 ▼

Signals to पहचान (identify) this pattern:

  • ✅ "Rotate array" / "Shift elements" / "Circular shift"
  • ✅ "In-place" constraint with O(1) space
  • ✅ "Reverse a portion of array"
  • ✅ "Move last k elements to front" or vice versa

The reverse helper is used in:

  • Reversing words in a string ("hello world" → "world hello")
  • Next permutation algorithm
  • Palindrome checking
  • Any in-place rearrangement problem
🚫 12. When NOT to Use STEP 12 ▼
  • ❌ When you need to preserve the original array (reversal modifies in-place)
  • ❌ Linked list rotation — different approach needed (just move pointers)
  • ❌ When the "rotation" is actually a different operation (like circular buffer — use modular indexing instead)
💡 "In-place rearrangement chahiye? Reverse is your best friend. Three reverses = any rotation."
